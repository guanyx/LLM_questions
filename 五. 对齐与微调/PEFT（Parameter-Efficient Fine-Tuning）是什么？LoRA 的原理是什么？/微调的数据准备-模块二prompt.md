# 微调的数据准备：图 2 Prompt

Create a highly detailed vertical technical infographic in refined hand-drawn editorial sketch style on pure white background. The theme is "微调数据要准备多少？先别追求精确，先判断量级". The infographic should feel strategically useful, visually sophisticated, and immediately scannable. It should preserve the information density of a decision framework, while presenting the content with deliberate hierarchy, composition, and visual rhythm.

**Design Concept:**
Use a clean vertical editorial infographic with a strong central progression spine. The whole composition should feel like a premium planning dashboard for fine-tuning data estimation: structured, precise, elegant, and practical.

The reader should scan from top to bottom and complete one continuous planning flow:

- Step 1: score the task using four dimensions
- Step 2: map the total score to a starting data-volume range
- Step 3: ground the abstract score in concrete task-type recommendations

Do not make it look like a plain worksheet. Introduce subtle visual metaphors inside each module:

- Module 1 feels like a scoring console / evaluation panel
- Module 2 feels like a staircase estimator / range ladder
- Module 3 feels like a practical field guide / recommendation board

The whole image should feel like a compact but premium estimation framework for real-world fine-tuning planning.

**Layout Structure:**

**Top Section (Title Area - 10%):**

- Main title in bold, large font: "微调数据要准备多少？先别追求精确，先判断量级"
- Subtitle below in smaller type: "用一个简单评分法，快速估出第一批该准备几百、几千还是几万"
- Add a subtle top header system:
  - left corner micro label: "Data Sizing Framework"
  - right corner micro label: "Step 4"
- Typography should feel like a magazine cover mixed with a technical whitepaper
- Keep large breathing room around the title while maintaining strong information density below

**Body Structure (Main Content - 85%):**
Create three vertically stacked information modules connected by a slim central guide line or directional spine. Each module should feel visually distinct, but still part of one coherent system.

- Use oversized section numbers 01 / 02 / 03 as anchor elements
- Use asymmetric layout rhythm: title block on one side, content block on the other, alternating lightly for sophistication
- Use divider bands, arrow cues, and panel framing to create progression
- Keep information dense, but introduce visual breathing through grouping and hierarchy

## Module 1 - 评分维度

- Module title: "给 4 个维度打分，每项 1～3 分"
- Intro annotation: "现实里最重要的不是一开始精确到“多少条”，而是快速判断第一批数据应该落在哪个量级。"
- Visual idea: this section should feel like a scoring console or evaluation dashboard
- Show four refined scoring cards, each as its own compact panel:
  - 任务复杂度
    - 1 分：风格 / 格式
    - 2 分：明确任务，如抽取 / 摘要
    - 3 分：复杂生成 / 多轮决策
  - 场景分散度
    - 1 分：输入很集中
    - 2 分：有几类典型场景
    - 3 分：输入非常发散
  - 稳定性要求
    - 1 分：演示可用
    - 2 分：内部试用
    - 3 分：线上高稳定
  - 基座能力差距
    - 1 分：基座本来就会
    - 2 分：能做但不稳
    - 3 分：明显不擅长
- Add one emphasis note below:
  - "这是起步估算，不是精确公式。"
- Visual instruction:
  - Each scoring card should preserve all 1/2/3 levels clearly
  - Use compact gauge-like markers, vertical ticks, score chips, or mini dial motifs
  - Cards should feel like a professional evaluation interface, not simple bullets
  - The reader should immediately understand: this section is about structured judgment

## Module 2 - 总分对应起步量

- Module title: "打完分后，看总分落在哪一档"
- Intro annotation: "把抽象判断，落到一个可执行的第一批数据区间。"
- Visual idea: this section should feel like a staircase estimator, tier ladder, or vertical range scale
- Show four data-volume tiers with strong visual separation:
  - 4～5 分：先做 300～1000 条
  - 6～8 分：先做 1000～5000 条
  - 9～10 分：先做 5000～10000 条
  - 11～12 分：按 1 万条以上规划，并准备更严格评测
- Add one emphasis note:
  - "这不是科学公式，但非常适合接需求后的第一轮规划。"
- Visual instruction:
  - Use a step-up diagram, segmented range ladder, or vertical tier bars
  - Make the “起步量级” the visual focal point
  - Use gradual visual escalation from hundreds → thousands → tens of thousands
  - The viewer should feel the rising planning weight as scores increase

## Module 3 - 对应的快速建议表

- Module title: "不同目标下，第一批数据怎么准备"
- Intro annotation: "有了评分区间之后，还要落回到具体任务类型，才能真正开始准备数据。"
- Visual idea: this section should feel like a practical field guide or recommendation board
- Show a refined four-column recommendation table:
  - Column A: 目标类型
  - Column B: 数据
  - Column C: 起步
  - Column D: 风险
- Populate the table with all original content:
  - 风格对齐 → 高质量问答对 → 300～1000 → 风格一致但业务正确性未必提升
  - 格式约束 → 输入到标准结构化输出 → 500～2000 → 边界输入覆盖不足时容易漏字段
  - 分类 / 抽取 / 摘要 / 改写 → 单任务监督样本 → 1000～5000 → 标注口径不一致
  - SQL / 代码 / 复杂生成 → 需求到目标输出，包含难例 → 3000～10000 → 只覆盖简单样本，线上复杂 case 崩
  - 行业问答 → 问题 + 上下文 + 回答 → 若无检索，慎重微调 → 把知识更新问题误判成微调问题
- Add one emphasis note below:
  - "这个表的作用，不是替你精确算出样本量，而是让你先快速落到一个合理区间。"
- Visual instruction:
  - Use a highly readable table-like layout with elegant row grouping
  - Make the “起步” column visually prominent through color, weight, or badge treatment
  - Use muted alert styling for the “风险” column so it reads practical but not alarming
  - Avoid making this look like a plain spreadsheet; it should feel curated and editorial
  - This section should visually communicate: "now the estimate becomes operational"

**Bottom Section (Summary Area - 5%):**

- Closing summary in 3 short lines:
  - "先判断量级"
  - "再映射到任务类型"
  - "最后再准备第一批数据"
- Final line:
  - "先把量级判断对，再进入训练和扩容。"
