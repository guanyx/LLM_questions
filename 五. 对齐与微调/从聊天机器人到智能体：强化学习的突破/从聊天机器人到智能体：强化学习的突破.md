# 从 Next-Token Prediction 到 Reinforcement Learning：Agent 范式转移的工程视角

在 2023-2024 年的大模型技术栈中，**SFT（Supervised Fine-Tuning）** 是绝对的主流。通过模仿高质量的人类对话数据，我们成功让模型掌握了语言的统计规律。

然而，进入 2025 年，随着 DeepSeek-R1、OpenAI o1/o3、Minimax M2.5 等模型的发布，技术风向标已经明确指向了 **RL（Reinforcement Learning）**。对于工程师而言，这不是简单的“微调升级”，而是一个从**“拟合数据分布”**到**“搜索与规划”**的底层范式转移。

本文将剥离市场营销术语，从技术原理、架构设计和工程挑战三个维度，深度解析这一轮 RL 浪潮。

---

## 一、 为什么 SFT 撞墙了？（The Theoretical Limit）

在理解 RL 之前，必须先明确 SFT 的局限性。
SFT 的本质是 **Behavior Cloning（行为克隆）**。它的目标函数是最大化给定上下文下，人类标注数据的似然概率（Likelihood）。

$$ \mathcal{L}_{SFT} = - \sum \log P(token_t | context, token_{<t}) $$

这就带来了两个无法通过堆数据解决的工程问题：

1.  **缺乏 Lookahead（前瞻性）与 Backtracking（回溯）：**
    SFT 模型是贪婪的（Greedy）。它每生成一个 Token，都是基于当前的局部最优解。
    - _Bug 场景：_ 写代码时，第一行写错了一个变量名，SFT 模型往往会试图“圆”回来，而不是删掉重写。因为它没有 **MCTS（蒙特卡洛树搜索）** 那样的回溯机制。
2.  **数据分布的“天花板”：**
    人类的思维过程是隐性的。我们在解决复杂数学题或 Coding 问题时，草稿纸上留下的只是**结果**，而不是脑海中尝试、推翻、再尝试的**思维链（Chain of Thought）**。SFT 只能学到结果，学不到过程。

**RL 的核心价值在于：** 它不再模仿人类的“输出”，而是通过定义 **Reward Function（奖励函数）**，让模型在巨大的解空间（Solution Space）中通过试错（Trial and Error）**搜索**出比人类演示更好的路径。

## 二、 DeepSeek-R1 与 OpenAI o3：System 2 思维的涌现

这两家公司的核心突破在于利用 RL 激发模型的推理能力（Reasoning），即 System 2 思维。

### 1. 技术核心：Test-Time Compute（推理时计算）

OpenAI o1/o3 和 DeepSeek-R1 的本质区别在于，它们把一部分训练算力转移到了推理阶段。模型在输出最终答案前，会生成大量的**隐式思维链（Hidden CoT）**。

### 2. 算法创新：GRPO (DeepSeek) vs PPO

DeepSeek-R1 的成功不仅在于数据，更在于算法效率。传统的 **PPO (Proximal Policy Optimization)** 需要一个与 Policy 模型等大的 Critic 模型（Value Function），这在显存上是不可接受的（相当于双倍参数量）。

DeepSeek 提出的 **GRPO (Group Relative Policy Optimization)** 做了一个关键的工程优化：

- **去掉了 Critic 模型：** 直接从同一个 Prompt 生成一组输出（Group of Outputs）。
- **组内相对奖励：** 以这组输出的平均得分作为 Baseline。
  $$ A_i = \frac{R_i - \text{mean}(R)}{\text{std}(R)} $$
- **Why it works:** 这极大地降低了 RL 的训练成本，使得在超大模型上进行 RL 成为可能。这也解释了为什么 DeepSeek 能以极低的成本复现 o1 的效果。

### 3. “Aha Moment” 与 Pure RL

DeepSeek-R1-Zero 的实验表明，即使没有 SFT 冷启动数据，仅靠纯 RL 和一个简单的验证器（Verifier，如编译器或答案检查器），模型也能涌现出自我反思能力（Self-Correction）。

- _现象：_ 模型会在思维链中输出 `<think> wait, this approach is wrong... </think>`。
- _工程启示：_ 推理能力可能不是被“教”出来的，而是被 RL 的搜索压力“逼”出来的。

## 三、 Minimax M2.5 与 Anthropic：Agent 环境交互

如果说 DeepSeek 是在“脑”内进化，Minimax 和 Anthropic 则是专注于“手”的进化——即 **Tool Use** 和 **Environment Interaction**。

### 1. 为什么 Agent 需要 RL？

在 Agent 场景（如操控电脑、运维服务器）中，SFT 最大的问题是 **Error Accumulation（误差累积）**。一步操作失误，后续所有操作都基于错误状态。

### 2. Minimax 的架构师直觉：Curriculum Learning（课程学习）

Minimax M2.5 在 RL 训练中引入了大规模的模拟环境。

- **技术点：** 他们构建了从文本游戏到复杂 IDE 的数十万个 Environment。
- **Breakthrough：** 通过 RL，模型学会了**“先规划（Spec），再执行”**的策略。这并非硬编码的规则，而是模型在无数次任务失败后，发现“写文档”能显著提高后续代码生成的 Reward，从而习得的**最优策略（Optimal Policy）**。

### 3. Anthropic Computer Use：像素级 RL

Claude 的 Computer Use 实际上是一个 **Pixel-to-Action** 的 RL 问题。

- **挑战：** Action Space 极大（屏幕分辨率 x 键鼠操作组合）。
- **解决方案：** 类似于 AlphaStar（星际争霸 AI），通过 RLAIF（RL from AI Feedback）和大规模截图数据，训练模型理解 GUI 的状态变化。它不再预测下一个 Token，而是预测下一个 `(x, y)` 坐标和 `click` 事件。

## 四、 GLM-5：超大模型 RL 的工程架构

当模型参数达到 100B+ 级别时，RL 的训练效率成为最大瓶颈。RL 需要模型不断生成（Rollout），这比单纯的 Backprop 慢得多。

### 1. 异步 RL 架构（Asynchronous Architecture）

GLM-5 采用了类似于 **Slime** 的架构，将训练过程解耦为：

- **Actor (CPU/GPU 集群)：** 负责与环境交互，生成轨迹（Trajectories）。
- **Learner (GPU 集群)：** 负责从 Replay Buffer 中采样，更新模型参数。

### 2. 为什么需要解耦？

- **Throughput（吞吐量）：** 生成数据（Inference）的速度通常远慢于训练（Training）。通过异步架构，可以由成千上万个 Actor 并发生成数据，喂给少量的 Learner，最大化 GPU 利用率。
- **On-Policy vs Off-Policy：** 虽然 PPO 是 On-Policy 的，但在大规模工程实现中，通常会引入一定的 Off-Policy 修正（如 V-trace），以容忍 Actor 和 Learner 之间的参数滞后（Lag）。

---

## 五、 总结：从“概率统计”到“价值对齐”

对于开发者而言，2025 年的模型选型和应用开发需要转变思维：

1.  **Prompt Engineering -> Reward Engineering：**
    以前我们调优 Prompt，以后我们可能更多地是在定义 Reward Function（如何判断任务做好了？）。
2.  **Inference Only -> Inference + Search：**
    不要指望模型一次输出完美答案。给模型“思考时间”，或者在应用层实现 Best-of-N 的采样和验证，本质上是在应用层做推理时的 RL。

**结论：** 强化学习的突破，本质上是让大模型从**“只会预测下一个字”**的文科生，进化成了**“懂得为了最终目标去搜索、规划、反思”**的理科生。这才是通往 AGI 的必经之路。
