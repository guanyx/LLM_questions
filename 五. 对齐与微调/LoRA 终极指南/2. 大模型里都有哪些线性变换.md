# 大模型用到了线性代数里的哪些线性变换？以及它们分别出现在哪里

理解“大模型里有哪些线性层”只是第一步。更有用的一种视角是反过来问：线性代数里那些经典的线性变换，在大模型训练与推理的哪些环节被反复用到？

你会发现一个规律：模型结构图看起来很花，但背后不断出现的其实是同一批“线性代数动作”——只是在不同位置、不同尺度、不同约束（低秩、稀疏、共享、结构化）下反复复用。

下面先给出一个“变换清单”，再把每个变换映射到大模型里的具体落点。

---

## Step 0：先把“线性变换”说准确

本文的“线性变换”指的是：满足叠加性与齐次性的一类映射，工程里最常见的表现是“矩阵乘向量/矩阵”。很多深度学习实现会带偏置项（严格来说是仿射变换），但不影响我们讨论“线性代数动作”在模型里的分布。

同时要强调一个边界：注意力、归一化、激活函数里含有非线性步骤。本文只聚焦它们里面的线性部分，以及训练/推理流程里用到的线性代数工具。

---

## Step 1：线性代数变换清单（大模型里反复出现的那一批）

你可以把下面每一条当作“在模型图上寻找的目标”，看到一个模块就去对号入座。

### 1) 一般线性映射（General Linear Map）

直白定义：把一个向量“重新表达一遍”。输出向量的每一维，都是输入各维度按不同权重做的加权求和。

数学上的通俗表述：你可以把它理解成“把输入向量丢进一排可学习的加权求和器”。矩阵的每一行决定一个输出维度怎么从输入各维度取权重、怎么混合。它可以改变维度大小（升维/降维），也可以在同维度内换一套坐标系（重编码）。

在大模型中的作用：

- **重编码**：把同一段信息换一种更适合后续模块处理的表示方式。
- **通道混合**：让不同特征维度彼此“交流”，而不是各干各的。
- **升维/降维**：在算力与表达能力之间做取舍（例如先扩张到更宽的中间空间再压回去）。

在大模型里出现在哪：

- LLM/Transformer：注意力的 Q/K/V/O 投影、FFN 的上/下/门控投影、输出头
- ViT：patch embedding（把局部图像块映射到 token 向量空间）、后续 Transformer 的各类投影
- 多模态：把不同模态特征映射到可对齐或可交互的公共维度

### 2) 仿射变换（Affine Transform）

直白定义：在线性映射的基础上，再加一个“基线偏移”。如果说线性映射负责“怎么混合输入”，偏置负责“把结果整体往哪边挪一挪”。

数学上的通俗表述：把线性层写成 “输出 = 矩阵 × 输入 + 偏置”。矩阵决定“按什么权重把输入混起来”，偏置决定“输出每个维度从什么起点开始”。没有偏置时，输入为 0 就只能输出 0；有偏置时，输入为 0 也能输出非 0。

在大模型中的作用：

- **提供默认基线**：让模型能表达“即使证据很弱也有默认倾向”的行为。
- **补偿分布偏移**：把输出整体平移到更合适的数值区间，便于后续非线性/门控工作。
- **配合归一化取舍**：若后面紧跟归一化，偏置的“平移”可能被抵消，因此工程上常选择省略或保留（看架构与实现习惯）。

在大模型里出现在哪：

- 线性层是否带 bias（常见是投影层/输出层）
- 归一化层的可学习仿射参数（常写成 γ、β 或 scale/shift）

### 3) 内积 / 点积（Inner Product / Dot Product）

直白定义：一种“相似度/对齐程度”的计算方式。两个向量越同向，点积越大；越反向，点积越小。

数学上的通俗表述：点积就是“把两个向量对应维度逐一相乘再相加”。你可以把它看成：一个向量在“问问题”，另一个向量在“给答案”，相乘相加得到一个分数，分数越大代表越匹配。

在大模型中的作用：

- **做匹配打分**：把“哪个更相关”变成可计算的分数。
- **把注意力变成检索**：注意力的核心就是大量点积打分，再把分数变成权重去加权汇总信息。
- **形成可训练的相似度空间**：在检索/对比学习里，点积（或其归一化版本）就是“距离/相似度”的主力算子之一。

在大模型里出现在哪：

- 注意力打分：Query 与 Key 的点积产生相关性分数
- 检索/对齐：文本-图像、文本-文本等表征之间的相似度计算
- 路由/匹配：例如选择专家、选择候选项时的打分步骤（视具体架构而定）

### 3.5) 外积（Outer Product）与秩一更新（Rank-1 Update）

直白定义：外积把两个向量“拼”成一个矩阵：一个向量决定行方向，一个向量决定列方向。秩一更新就是“用一个外积给矩阵加一块最简单的改动”。

数学上的通俗表述：外积可以理解成“把一个向量复制成很多行/列，再按另一个向量逐行/逐列缩放”，得到一个结构非常简单的矩阵。这个矩阵能表达的独立变化方向只有一条，所以叫“秩一”。

在大模型中的作用：

- **理解训练信号怎么写进参数**：很多线性层的梯度形态都能直观地理解成“输入信号 × 误差信号”的外积累加。
- **解释结构化更新的来源**：很多高效更新（包括低秩更新的直觉）都可以看成“很多个秩一更新叠起来”。

在大模型里出现在哪：

- 反向传播时的权重梯度（尤其是线性层/卷积层的权重更新形式）
- 优化器在更新参数与状态时的大量线性组合（虽然实现里不一定显式写成“外积”）

### 4) 投影（Projection）与子空间选择（Subspace Selection）

直白定义：只保留信息的一部分“方向”。投影相当于挑选一些方向保留，其它方向丢弃或弱化。

数学上的通俗表述：把一个向量拆成“在某个空间里那一部分 + 不在那个空间里的那一部分”，投影只拿前者。很多时候你看到的“先压缩再还原”，本质就是“先投影到低维子空间，再从子空间映回去”。

在大模型中的作用：

- **筛选与压缩**：把信息集中到更少的维度里，让后续计算更省、更稳。
- **并行学习不同关系**：把表示拆成多个子空间分别处理（典型就是多头注意力的“分头”）。
- **把同一信息换成更适合交互的形态**：例如跨模态对齐时，把不同模态投到一个可比较/可交互的子空间里。

在大模型里出现在哪：

- 各类投影矩阵（Projection/Linear）的“映射到某个空间”
- 多头注意力的 head 切分与合并（把表示分到多个子空间里并行处理）
- 各种瓶颈结构（降维 → 升维）的压缩与还原路径

### 5) 低秩分解（Low-Rank Factorization）

直白定义：用更少的“独立方向”去表达一个矩阵。你限制这个变换最多只能沿着少数几条主方向去改变输入。

数学上的通俗表述：把一个大矩阵拆成两个更小的矩阵相乘（常见形式是“先降维再升维”）。这等价于：你只允许模型在一个较小的中间空间里做变化，再把变化投回原空间。

在大模型中的作用：

- **参数高效微调**：用很少的可训练参数，改变模型在少数关键方向上的行为（LoRA 的核心直觉）。
- **降低存储与带宽压力**：更新量更小，训练/部署更省。
- **引入结构化约束**：更不容易把基座模型能力“全局扰乱”，但对需要大幅改动的任务可能有上限。

在大模型里出现在哪：

- LoRA：对线性层权重的增量采用低秩形式
- Adapter/瓶颈模块：降维 → 非线性 → 升维（线性部分就是低秩瓶颈）
- 一些压缩与加速方案中的低秩近似（是否使用取决于系统与目标）

### 6) 旋转 / 正交变换（Rotation / Orthogonal Transform）

直白定义：一种“只换坐标系、不改几何形状”的变换。它改变坐标表示，但不改变长度与夹角（因此很多相似度关系保持不变）。

数学上的通俗表述：你可以把它理解成“把坐标轴整体转一下”。向量的数值会变，但向量之间的相对几何关系（距离/夹角）保持不变。

在大模型中的作用：

- **保持相似度结构**：当你希望“相似的仍然相似”时，旋转/正交是天然选择。
- **更自然地注入结构信息**：某些位置编码用“在二维子空间旋转”的方式把位置信息折进点积里，让注意力打分天然带上相对位置信息。

在大模型里出现在哪：

- 旋转式位置编码一类设计（把位置信息写进表示的几何关系里）
- 一些初始化/约束里追求正交性（不一定显式出现为一个“旋转层”）

### 7) 对角缩放（Diagonal Scaling）与逐通道重标定

直白定义：对每个维度单独乘一个系数，只调强弱，不做维度间混合。

数学上的通俗表述：等价于乘以一个“只有对角线有数、其他位置全是 0”的矩阵。每个维度都有自己的倍率，彼此互不干扰。

在大模型中的作用：

- **细粒度调节重要性**：哪些特征该放大、哪些该压小，用很少参数就能做到。
- **稳定训练**：作为归一化/门控的一部分，帮助控制数值尺度与梯度流动。

在大模型里出现在哪：

- 归一化层的缩放参数（常写作 γ 或 scale）
- 各类门控里的逐元素乘法（对不同维度做重权重）

### 8) 置换（Permutation）与块重排（Block Reshaping）

直白定义：把数据“换个排列方式”，但不改变数值本身。每个输出位置只是把某个输入位置搬过来。

数学上的通俗表述：在矩阵语言里，这等价于乘以一个“每行每列只有一个 1、其他都是 0”的置换矩阵；它不做加权求和，只做位置重排。

在大模型中的作用：

- **分组并行处理**：例如把隐藏维度切成多个头并行做注意力。
- **匹配高效实现的内存布局**：为了更快的矩阵乘法/融合算子，需要把张量维度重排到更合适的排布。

在大模型里出现在哪：

- 多头注意力的 split/merge（拆头与合头）
- 张量并行/算子融合/内核优化中的维度重排与分块

### 8.5) 转置（Transpose）与对称化视角

直白定义：把矩阵的“行和列”对调。

数学上的通俗表述：如果把矩阵看成“输入到输出的线性关系表”，转置就是把这张表沿对角线翻一下，让“行的角色”和“列的角色”互换。很多时候它等价于一次特殊的重排。

在大模型中的作用：

- **服务高性能矩阵乘法**：为了匹配某些高效内核的布局需求，经常需要转置或等价的布局变换。
- **让推导更简洁**：在公式里转置常用来表达“把映射方向换一下”或“把两个向量点积写成矩阵形式”。

在大模型里出现在哪：

- 注意力、并行与块矩阵乘法的实现细节里（常被融合进 reshape/permute 或更换布局）
- 论文推导里大量出现的 “^T” 符号（尤其是点积、低秩与矩阵分解相关表达）

### 9) 卷积作为结构化线性算子（Convolution as a Structured Linear Operator）

直白定义：一种带强约束的线性映射：每个输出位置只看输入的局部邻域，而且同一套权重在所有位置共享。

数学上的通俗表述：卷积本质上就是“在每个位置做一次局部加权求和”。因为它只做加权求和，不引入“按输入决定系数”的动作，所以它是线性的；它之所以特别，是因为“局部连接 + 权重共享”的结构约束。

在大模型中的作用：

- **把视觉/时空数据的局部结构写进模型**：更符合图像/视频的局部性与平移等变特点。
- **用结构约束换效率与泛化**：比全连接更省参数、更易优化，同时提供强归纳偏置。
- **做通道混合的高效实现**：1×1 卷积常用于把通道维度当作“线性投影”来混合。

在大模型里出现在哪：

- 扩散模型 U-Net 的主干卷积块（参数与算力集中区之一）
- 视觉编码器的前端与 patch embedding（有时用卷积实现“切块+投影”）

### 10) 稀疏/块稀疏/块对角（Sparse / Block-Sparse / Block-Diagonal）

直白定义：让“本来可能发生的连接/相互作用”变少：只允许一部分位置相互作用，其余位置固定为 0 或被屏蔽。

数学上的通俗表述：把一个原本“密密麻麻都有数”的矩阵，强行改成“只有少量位置有数”的矩阵，或只保留若干块。它依然是线性变换，只是连接模式被结构化地限制住了。

在大模型中的作用：

- **把计算量降下来**：尤其在长上下文里，减少不必要的相互作用能显著降低计算与访存压力。
- **把架构假设写进模型**：例如“更重视局部邻近”“分块内交互更强”等。
- **支撑稀疏激活路线**：例如只让 token 经过少数专家，减少整体计算（MoE 的常见动机之一）。

在大模型里出现在哪：

- 长上下文注意力的局部/滑窗/分块等模式（是否使用取决于模型与系统）
- MoE 的路由选择（每个 token 只走少数专家，体现出“稀疏使用”的结构）

### 11) 张量积 / Kronecker 结构（Tensor Product / Kronecker-Structured Maps）

直白定义：把一个“大变换”拆成多个“小变换的组合”，并且组合方式很规则，从而显著减少参数与计算。

数学上的通俗表述：把一个大矩阵写成若干个小矩阵按某种规则“拼出来”（张量积/Kronecker 结构就是其中典型）。直觉上相当于：你不让变换在所有维度上随意混合，而是按“空间 × 通道 × 时间”等可分解维度分别处理，再按规则合成。

在大模型中的作用：

- **用规则换效率**：牺牲一部分自由度，换来更少参数、更快计算或更好可扩展性。
- **对多维数据更友好**：在图像/视频/多模态场景里，更容易把“维度的天然分解结构”利用起来。

在大模型里出现在哪：

- 一些参数高效或压缩型层设计（工具箱性质，不是所有主流模型必备）
- 多维数据的结构化线性层设计中（用来分解空间/通道/时间等维度的线性关系）

## Step 2：把“变换清单”映射回模型：你在结构图上该怎么找

如果你拿到一张模型结构图，建议按下面顺序扫描：

1. 先找所有“矩阵乘法”密集区：通常就是投影矩阵与卷积块
2. 再找“相似度/匹配”算子：通常就是点积（以及它后面的归一化）
3. 最后找“结构约束”：低秩、共享、稀疏、分块、置换

用这三个步骤，你能把任何一个大模型快速翻译成“线性代数动作序列”。

为了更具体，下面给出三类典型模型的落点地图。

### A) LLM/Transformer（文本为主）

- 一般线性映射：注意力 Q/K/V/O、FFN 的上/下/门控投影、输出头
- 内积：注意力打分、检索/对比式匹配（若有）
- 投影与子空间：多头注意力（多子空间并行）、FFN 的扩展与压缩
- 旋转/正交：旋转式位置编码把位置信息折进相似度计算
- 对角缩放：归一化的 γ、门控的逐维重权重
- 置换/重排：多头切分与拼接、并行计算的布局变化
- 低秩：LoRA 与各类低秩瓶颈式适配
- 稀疏/块稀疏：长上下文注意力的一类常见加速路线（是否使用取决于模型与系统）

### B) ViT/VLM（视觉与多模态）

- 一般线性映射：patch embedding、跨模态对齐投影、cross-attention 的 Q/K/V/O
- 内积：对齐与相似度（对比学习、检索）、跨注意力匹配
- 卷积作为结构化线性：视觉前端或 patch embedding 的实现方式之一
- 投影与子空间：多头注意力与多层融合，本质仍是“换表示空间再交互”

### C) 扩散模型 U-Net（图像生成）

- 卷积作为结构化线性：主干大量卷积层（参数与算力集中区）
- 一般线性映射：通道混合的 1×1 卷积可视作“逐位置线性投影”
- 内积与投影：当模型引入注意力块时，回到注意力的 Q/K/V/O 与点积匹配

## Step 3：三个常见误区（很容易把线性代数看“偏”）

1. “注意力就是线性变换”  
   注意力里确实充满线性投影与内积，但注意力的核心步骤包含归一化与加权聚合等非线性/数据依赖过程。把它完全当线性会误判它的表达能力与稳定性来源。

2. “只要懂矩阵乘法就懂大模型”  
   矩阵乘法是大头，但大模型的行为往往由“线性变换 + 非线性 + 训练目标 + 数据分布”共同塑形。只看线性层会忽略对齐、泛化与安全等关键问题。

3. “低秩/稀疏一定更好”  
   低秩与稀疏是结构约束：它们能省资源，也会限制表达自由度。是否更好取决于任务、数据、预算以及你愿意承担的工程复杂度。

---

## Step 4：现实取舍：为什么这些变换会成为工程与研究的主战场

- 线性算子容易加速：硬件对矩阵乘法/卷积有成熟支持，系统优化也容易围绕它做融合、并行、量化
- 线性算子易做结构约束：低秩、共享、稀疏、分块能以明确方式约束参数与计算
- 瓶颈随规模变化：模型越大、上下文越长，越需要用“结构化线性代数”去交换算力、带宽、显存与精度

前沿变化（近 12–24 个月常见讨论点之二与之三）：

- 混合精度与更低比特的量化在推理端普及，线性层的数值表达与舍入误差处理成为关键
- 长上下文推动注意力的分块/稀疏/近似路线加速落地，很多变化不是“换了模型思想”，而是“换了线性代数的计算方式”
