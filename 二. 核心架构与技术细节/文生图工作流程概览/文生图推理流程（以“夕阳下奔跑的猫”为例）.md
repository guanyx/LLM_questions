# 文生图推理流程（以“夕阳下奔跑的猫”为例）

目标：不用一上来就讲一堆抽象术语，而是直接跟着一个真实示例走一遍。假设用户输入：

> 一只在夕阳下奔跑的猫，金色逆光，草地，动态感，写实风格

不管底层是扩散模型、流匹配模型，还是更具体的某个产品实现，现代文生图系统通常都会经过一条相似的主链路：`理解这句话是什么意思 -> 在潜空间里从噪声开始逐步成形 -> 解码成可见图片 -> 做必要的后处理并返回结果`。下面就按这个顺序，把“第一张图是怎么出来的”讲清楚。

---

## 1. 用户这句话，先被变成模型能处理的输入

模型并不能直接“读懂中文句子”。它首先要把这句话变成一串内部符号。

1. Prompt 预处理：系统会先清理空格、统一一些写法，必要时处理权重语法或模板片段。这样做不是为了让句子更好看，而是为了减少“同一个意思，换个写法结果差很多”的不稳定。就像把口头需求整理成格式统一的需求单。
2. 分词与切 token：`一只`、`夕阳`、`奔跑`、`猫`、`写实风格` 这些内容，会被切成模型词表里的 token。这里的 token 不一定等于自然语言里的“一个词”，更像是一组高频语义碎片。
3. 映射成 token ID：每个 token 再被换成整数编号，方便后面的神经网络查表和计算。到这一步为止，模型还没有“画画”，它只是把一句人话变成了机器可处理的离散输入。

可以把这一步理解成：用户给的是一句创意描述，系统先把它翻译成一套内部工单编号。

---

## 2. 模型开始理解：这句话到底想生成什么

接下来，文本会进入文本编码器。它的任务不是直接出图，而是把这句话压成一份“语义说明书”。

1. 编码关键信息：模型会逐步捕捉到几个核心要求。
   - 主体是什么：一只猫
   - 动作是什么：在奔跑
   - 环境是什么：草地、夕阳
   - 氛围是什么：金色逆光、动态感
   - 风格是什么：写实
2. 形成条件表征：编码器输出的不是一句“中文解释”，而是一组向量表征。它们会告诉后面的图像生成网络，哪些词彼此相关，哪些特征应该同时出现。比如“猫”和“奔跑”会约束身体姿态，“夕阳”和“金色逆光”会约束光线方向与整体色调。
3. 可选的负面条件：很多系统还会同时准备一份“不想要什么”的条件，比如不要模糊、不要多只猫、不要畸形肢体。这样在后面做 guidance 时，可以把“想要”和“不想要”拉开距离。

这一步像什么？像创意总监把一句模糊需求，拆成了一张更精确的视觉 brief：主角、动作、场景、光线、风格，都先定义清楚。

---

## 3. 真正开始“画”之前，系统先准备一块看不见的画布

这里有一个非常反直觉的点：文生图模型通常不是在像素空间里直接从空白画布开始画，而是在潜空间里操作。

1. 先生成一团随机噪声：系统会根据一个 seed 采样出初始 latent。它看起来不是“模糊的猫”，而是没有人类可读意义的一团随机状态。
2. 确定输出尺寸：比如你想要横图、竖图，还是某个固定分辨率，系统会据此决定潜空间的大小。最终想要的画布越大，内部要处理的 latent 通常也会相应变化。
3. 设定去噪时间表：现代系统通常不会用特别多的步数，而是预先规划好一条从“噪声很大”逐步走到“噪声接近零”的路径。工程上常用连续时间或噪声强度 `sigma` 来描述当前阶段。
4. 选择采样器或求解器：不同求解器决定“每一步怎么走”。目标是在尽量少的步数里，既保住结构，又别把画面带崩。近年的主流趋势就是更少步采样，以及配合蒸馏、一致性方法来加速。

这一步像是正式开画前的准备工作：先选画布大小，再拿出一张完全涂满噪点的底稿，然后决定要分几轮把它一点点修成画面。

---

## 4. 采样开始：模型不会一次出图，而是很多小步逐渐成形

真正的生成过程，通常是一个循环。每一步都回答两个问题：

- 现在这张“噪声中的图”处于什么阶段？
- 按照“夕阳下奔跑的猫”这个要求，下一步该往哪个方向修？

在每一个采样步里，系统通常会做这些事：

1. 取当前时间点或噪声强度：比如当前还很“糊”，说明噪声很大；如果已经接近成图，说明噪声已经比较低。
2. 准备当前 latent：有些采样器会对 latent 做缩放或归一化，避免不同噪声阶段的数值尺度差太多。
3. 带上文本条件一起送进去噪器：此时网络看到的不是单独的“噪声块”，而是“当前 latent + 当前阶段 + 文本条件”。也就是说，它同时知道“现在画到哪一步了”和“最终想画成什么”。
4. 如果启用 CFG，还会同时跑一份弱条件或无条件分支：这样后面就能把“自由生成方向”和“按 prompt 走的方向”做差，再控制 prompt 的约束力度。

要注意，模型并不是像人类画师那样先画猫头、再画腿、再画夕阳。它更像是在一张不断变化的隐空间草稿上，全局地同时调整主体、构图、颜色和纹理。

---

## 5. 去噪器在每一步里，到底干了什么

这一环是核心。现代文生图系统里，最关键的“画图大脑”通常就是去噪器，很多新系统里它往往是 Transformer 或 Diffusion Transformer 一类结构。

1. 接收三类信息：
   - 当前 latent：也就是“现在这张半成品长什么样”
   - 当前时间或 sigma：也就是“现在噪声还剩多少”
   - 文本条件：也就是“最终目标是夕阳下奔跑的猫”
2. 在网络内部融合条件：文本不会只在入口处看一眼就结束，它通常会通过 cross-attention、调制层等机制，持续影响中间特征。于是“猫”“奔跑”“夕阳”“写实”这些条件会不断参与每一步判断。
3. 预测修正方向：去噪器会输出一种关于“该怎么把当前状态往更清晰图像推进”的预测。不同模型实现里，这个预测可能写成噪声 `epsilon`、速度 `v`，或者更接近干净图像的 `x0`。名字不同，核心作用类似：给求解器一个更新方向。

如果把它类比成修图过程，去噪器并不是直接说“最终图长这样”，而是每一步都在说：“当前这团东西里，哪些部分更像猫，哪些部分更像背景，哪些颜色和光线方向更符合夕阳逆光，再往那个方向推一点。”

---

## 6. 为什么经过很多步后，真的会越来越像“那只猫”

因为每一步更新都不是乱改，而是在文本条件约束下，把 latent 从高噪声状态往低噪声状态推进。

1. guidance 合成：如果系统同时算了条件分支和无条件分支，就会在这里把两者按比例合成。比例大一点，图更听 prompt；比例太大，也可能更容易失真、过饱和、结构不自然。
2. 求解器更新一步：求解器根据去噪器给出的方向，把 latent 从当前 `sigma_k` 推到下一个 `sigma_{k+1}`。这一步不是“删掉一点噪声”那么简单，而是沿着一条学习到的轨迹移动。
3. 多轮重复：一开始，模型可能只知道“这里大概有一只动物，背景偏暖”；再往后，猫的姿态会更明确，草地和地平线更稳定，逆光位置更合理，毛发和轮廓也更像写实照片。

一个比较接近真实过程的理解是：

- 前期更像在确定大构图和大结构
- 中期更像在稳定主体、动作和光影关系
- 后期更像在补局部细节、质感和边缘

但这只是便于理解的描述，不代表模型内部真有一条严格的“先构图再细化”的硬分工。真实计算是高度并行、全局耦合的。

---

## 7. 从看不见的 latent，到真正看到一张 RGB 图片

当噪声已经降到足够低，系统手里拿到的是一个“可以解码”的潜表示，但它还不是普通图片文件。

1. latent 后处理：有些实现会在解码前把 latent 调回解码器习惯的数值范围，避免颜色和对比度异常。
2. 解码成像素：接下来，解码器会把 latent 还原成真正的 RGB 图像。很多系统会使用 VAE 一类解码器，把潜空间里的压缩表示展开成肉眼可见的图。
3. 得到第一张图：到这里，用户终于能看到一张具体结果，比如一只猫正从画面左侧冲向右前方，背景有金黄色夕阳，草地被逆光照亮，整体风格偏写实。

可以把 latent 理解成“压缩过的视觉草稿”，解码器则像是把这份压缩草稿完整冲洗成一张可查看的成片。

---

## 8. 为什么有时候它画得对，有时候又会跑偏

同样一句“夕阳下奔跑的猫”，模型也不保证每次都画得完全一样。原因通常不神秘，主要集中在下面几类：

1. 随机种子不同：起点噪声变了，最后收敛到的细节和构图也可能变化。
2. prompt 约束不够强：如果“奔跑”写得不够明确，模型可能给你一只站着或跳跃中的猫。
3. guidance 太强或太弱：太弱时不够听话，太强时又容易显得僵硬、过拟合描述。
4. 模型先验本身有限：模型知道“猫”“草地”“夕阳”常一起出现，但它并不真的理解物理世界。于是可能出现尾巴姿态别扭、腿部数量异常、逆光关系不自然等问题。
5. 步数与求解器选择不同：少步采样更快，但如果模型和求解器配合不好，也可能损失细节或稳定性。

这也是为什么现代工程里，大家会不断优化采样器、蒸馏方法、文本对齐和后处理流程。目标不是改变主链路，而是在同一条主链路上，让它更快、更稳、更听话。

---

## 9. 最后交付给用户的，不只是那张图片

在产品层面，一次文生图通常还会附带一些补充环节。

1. 后处理增强：比如做超分、细节增强、局部修复，让毛发、草地纹理、光斑更自然。
2. 安全与合规：系统可能会检查输出内容是否触碰安全策略，并在必要时添加标识或水印。
3. 保存元信息：常见会记录 seed、步数、采样器、尺寸、prompt 等信息，方便复现或继续微调。

所以从用户视角看，你像是“输了一句话，立刻得到一张图”；但从系统视角看，它其实经历了一整套条件理解、潜空间采样、去噪迭代、解码和交付流程。

---

## 10. 用一句话重新记住整条链路

如果你只想带走一个最核心的理解，可以记成这句话：

**文生图不是模型读完 prompt 后“一次性画出图片”，而是先把 prompt 变成条件，再从一团随机噪声出发，经过很多次受条件约束的更新，最后把稳定下来的潜表示解码成一张可见图片。**

放回我们的例子里，就是：

用户输入“夕阳下奔跑的猫” -> 模型把这句话理解成主体、动作、场景、光线、风格等条件 -> 从随机 latent 开始逐步去噪 -> 在每一步都朝着“更像那只猫”的方向修正 -> 最后解码出第一张具体图像。
