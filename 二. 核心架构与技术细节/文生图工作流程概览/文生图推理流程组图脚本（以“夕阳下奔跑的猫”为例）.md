# 文生图推理流程组图脚本（以“夕阳下奔跑的猫”为例）

## 使用说明

- 形式：知识型组图，建议 `4:5` 竖版，适合做封面 + 分镜长图轮播
- 张数：`1 张封面 + 8 张分镜`
- 视觉基调：科技感信息图，不要做成花哨插画；核心是“流程准确、标注清晰、图间不重复”
- 统一视觉母题：都围绕同一个示例 prompt 展开，即 `一只在夕阳下奔跑的猫，金色逆光，草地，动态感，写实风格`
- 统一约束：
  - 所有“箭头”都表示信息流或状态更新方向，不能画成随意装饰线
  - `Prompt -> token -> 条件表征 -> latent 迭代 -> 解码成图` 这条总链路必须前后一致
  - “文本编码器”不能直接连到“最终图片”，中间必须经过 latent 采样与解码
  - “去噪器”的输出应标成“修正方向/预测量”，不要误写成“直接输出图片”

## 全组图分工

- 封面：只建立总主题和总链路，不展开细节
- 分镜 1：只讲 prompt 如何变成 token 和 token ID
- 分镜 2：只讲文本编码器如何提炼主体、动作、场景、光线、风格
- 分镜 3：只讲 latent 初始准备，包括 seed、尺寸、sigma 时间表、采样器
- 分镜 4：只讲“采样循环”长什么样，不解释去噪器内部机理
- 分镜 5：只讲去噪器内部接收什么、输出什么
- 分镜 6：只讲 guidance 和求解器如何把 latent 一步步推进
- 分镜 7：只讲 latent 解码成 RGB 图片
- 分镜 8：只讲为什么会跑偏，并用一句话收束到“解码成图”

## 封面

### 封面标题

`从Prompt到生成图片，过程是怎么一步步出来的？`

### 封面副标题

`以“夕阳下奔跑的猫”为例，看懂 Prompt 到成图的完整推理链路`

### 封面画面构图

- 版式采用居中主视觉 + 下方总流程条
- 中央主视觉不是成品海报，而是“一只半透明的奔跑猫轮廓”叠在渐变噪声云上
- 背景左侧偏冷色噪点，右侧偏暖色夕阳色带，暗示“从噪声到成图”
- 底部放一条 4 段式总流程条，作为全篇导航

### 封面必备元素

- 一段简短 prompt 文本框
- 一团抽象 latent 噪声
- 一张小尺寸成品图缩略图
- 一条总链路导航条

### 封面建议标注

- Prompt：`夕阳下奔跑的猫`
- 导航条节点 1：`文本输入`
- 导航条节点 2：`语义理解`
- 导航条节点 3：`latent 采样`
- 导航条节点 4：`解码成图`

### 封面连线要求

- 导航条使用单向箭头：`文本输入 -> 语义理解 -> latent 采样 -> 解码成图`
- 中央主视觉只做氛围，不要再加复杂分支箭头

### 封面避免重复

- 封面不要展开 token、cross-attention、CFG、VAE 等术语细节

## 分镜 1：Prompt 如何变成模型输入

### 分镜 1 目标

把“人类语言”变成“模型可处理的离散输入”，只讲预处理、切 token、映射 ID。

### 分镜 1 画面构图

- 从左到右的三段流水线
- 左侧是原始 prompt 文本框
- 中间是被切开的 token 小块
- 右侧是整齐排列的整数 ID 方块

### 分镜 1 必备元素

- 原始 prompt：`一只在夕阳下奔跑的猫，金色逆光，草地，动态感，写实风格`
- token 示例块：`一只`、`夕阳`、`奔跑`、`猫`、`写实风格`
- token ID 示例：`1024`、`381`、`9088` 这一类抽象整数即可，不必对应真实词表

### 分镜 1 建议标注

- 步骤 1：`Prompt 预处理`
- 步骤 2：`切分为 token`
- 步骤 3：`映射成 token ID`
- 页脚结论：`这一步还没有开始“画图”，只是把人话变成机器输入`

### 分镜 1 连线要求

- 主线箭头只能是：`Prompt 文本 -> token 序列 -> token ID 序列`
- 可在“Prompt 预处理”旁加一条细标注：`统一写法，减少表达扰动`

### 分镜 1 避免重复

- 不要画文本编码器内部结构
- 不要提前出现图片、latent、解码器

---

## 分镜 2：文本编码器如何理解需求

### 分镜 2 目标

把 token/ID 变成“语义说明书”，强调这是条件表征，不是直接出图。

### 分镜 2 画面构图

- 左侧是 token ID 输入条
- 中间是一个“文本编码器”模块
- 右侧拆成 5 张语义卡片 + 1 张负面条件卡片

### 分镜 2 必备元素

- 文本编码器模块
- 五类语义卡片：
  - `主体：猫`
  - `动作：奔跑`
  - `环境：草地 / 夕阳`
  - `光线氛围：金色逆光 / 动态感`
  - `风格：写实`
- 可选负面条件卡片：`不要模糊 / 不要多只猫 / 不要畸形肢体`

### 分镜 2 建议标注

- 编码器输出：`条件表征（向量）`
- 侧边说明：`告诉后面的生成网络“该画什么、哪些特征要一起出现”`
- 负面条件说明：`在后续 guidance 中用于拉开“想要”和“不想要”`

### 分镜 2 连线要求

- `token ID -> 文本编码器 -> 条件表征`
- `条件表征` 再分叉连接到 5 张语义卡片
- `负面条件` 单独成一路，不要和正向条件混成一个框

### 分镜 2 避免重复

- 不要画 seed、噪声、去噪步数
- 不要把编码器直接连到最终图片

---

## 分镜 3：正式采样前，先准备 latent 画布

### 分镜 3 目标

讲清楚采样前的四个准备项：seed、输出尺寸、sigma 时间表、采样器。

### 分镜 3 画面构图

- 中央是一块“latent 画布”，显示成抽象噪声矩阵
- 四周四个准备项像仪表盘一样围绕它

### 分镜 3 必备元素

- 中央元素：`初始 latent（随机噪声）`
- 准备项 1：`seed`
- 准备项 2：`输出尺寸 / 宽高比`
- 准备项 3：`sigma 时间表`
- 准备项 4：`采样器 / 求解器`

### 分镜 3 建议标注

- `seed：决定随机起点`
- `尺寸：决定 latent 空间大小`
- `sigma：从高噪声走向低噪声`
- `采样器：决定每一步怎么更新`
- 页脚结论：`模型不是直接在像素画布上起稿，而是在潜空间里从噪声出发`

### 分镜 3 连线要求

- 四个准备项都用箭头指向中央 `初始 latent`
- `sigma 时间表` 旁边可画一条从高到低的曲线，并标 `高噪声 -> 低噪声`

### 分镜 3 避免重复

- 不要讲文本条件如何进入网络
- 不要讲 CFG 和去噪器内部结构

---

## 分镜 4：采样不是一次出图，而是一个循环

### 分镜 4 目标

只展示“采样循环”的流程骨架，让读者先理解它是 many-step iteration。

### 分镜 4 画面构图

- 采用环形或回路式流程图
- 中心放一个不断变化的 latent 缩略图
- 外圈放四个循环步骤

### 分镜 4 必备元素

- 当前步标识：`step k`
- 当前噪声阶段：`当前时间 / sigma_k`
- 当前 latent
- 文本条件输入
- 可选无条件分支占位

### 分镜 4 建议标注

- 步骤 A：`读取当前时间 / sigma`
- 步骤 B：`准备当前 latent`
- 步骤 C：`把 latent + 阶段 + 文本条件送入去噪器`
- 步骤 D：`得到更新方向，进入下一步`
- 中央说明：`每一步都在回答：现在画到哪了？下一步往哪修？`

### 分镜 4 连线要求

- 环路箭头必须闭合，表现 `step k -> step k+1`
- “文本条件”用一条侧向输入箭头接入去噪器
- 若画无条件分支，只标 `optional`，不要在本页展开 CFG 合成公式

### 分镜 4 避免重复

- 不要解释去噪器内部的 cross-attention
- 不要解释 guidance 比例如何调

---

## 分镜 5：去噪器到底看到了什么，又输出什么

### 分镜 5 目标

聚焦“画图大脑”本身，强调输入三要素和输出是“修正方向”。

### 分镜 5 画面构图

- 中间一个大模块：`去噪器`
- 左侧三路输入，右侧一路输出
- 模块内部可以画出简化的 attention/特征融合示意

### 分镜 5 必备元素

- 输入 1：`当前 latent`
- 输入 2：`当前时间 / sigma`
- 输入 3：`文本条件`
- 模块内部提示：`cross-attention / 条件调制`
- 输出：`预测修正方向`

### 分镜 5 建议标注

- 输入说明：`它同时知道“现在长什么样”“现在处于哪一步”“最终想画成什么”`
- 输出说明：`可表示为 epsilon / v / x0，名字不同，作用相近`
- 页脚结论：`去噪器不是直接吐出最终图，而是告诉求解器“下一步往哪推”`

### 分镜 5 连线要求

- 三路输入全部汇入 `去噪器`
- `去噪器 -> 预测修正方向`
- 内部文本条件到特征层的细箭头可标成 `持续影响中间特征`

### 分镜 5 避免重复

- 不要把“求解器更新 latent”也画成这个模块内部工作
- 不要直接出现最终 RGB 图

---

## 分镜 6：为什么会越来越像那只猫

### 分镜 6 目标

讲 guidance 和求解器如何共同把 latent 从高噪声推进到低噪声。

### 分镜 6 画面构图

- 左侧放“条件分支”和“无条件分支”两个小输出
- 中间是 `guidance 合成`
- 右侧是一条从 `sigma_k` 到 `sigma_{k+1}` 再到更低噪声的推进轨迹
- 底部加三阶段时间轴：前期 / 中期 / 后期

### 分镜 6 必备元素

- 条件分支输出
- 无条件分支输出
- `guidance scale`
- 求解器更新箭头
- 三阶段 latent 缩略图：粗结构、中间态、细节增强

### 分镜 6 建议标注

- `guidance：控制“有多听 prompt”`
- `求解器：按预测方向把 latent 推到下一时刻`
- `前期：先稳住构图和大结构`
- `中期：稳定主体、动作、光影关系`
- `后期：补局部细节、质感和边缘`

### 分镜 6 连线要求

- 两路分支先汇入 `guidance 合成`
- `guidance 合成 -> 求解器`
- `求解器 -> latent @ sigma_{k+1}`
- 时间轴方向必须与噪声下降方向一致，标成 `高噪声 -> 低噪声`

### 分镜 6 避免重复

- 不要再重新解释什么是 token、文本编码器
- 不要提前画解码器

---

## 分镜 7：latent 如何变成真正可见的图片

### 分镜 7 目标

专门讲解码，强调 latent 不是图片文件，必须经过解码器还原成 RGB。

### 分镜 7 画面构图

- 左侧是一块较为平滑的最终 latent
- 中间是 `解码器 / VAE Decoder`
- 右侧是一张清晰的成品图

### 分镜 7 必备元素

- `稳定后的 latent`
- `latent 数值范围调整`
- `解码器 / VAE`
- `RGB 图片`
- 成图示例：奔跑猫、金色夕阳、草地、写实质感

### 分镜 7 建议标注

- `latent：压缩过的视觉草稿`
- `解码前：调整到解码器习惯的数值范围`
- `解码后：展开为肉眼可见的 RGB 图像`
- 页脚结论：`用户真正看到的第一张图，是解码后的结果，不是 latent 本身`

### 分镜 7 连线要求

- `稳定后的 latent -> 数值调整 -> 解码器 -> RGB 图片`
- 可以在解码器旁边加一条小注：`把潜空间表示还原到像素空间`

### 分镜 7 避免重复

- 不要再出现采样循环结构
- 不要把超分、安全检测混进来

---

## 分镜 8：为什么有时会跑偏

### 分镜 8 目标

只解释为什么同一句 prompt 有时画得准、有时会跑偏，并把整条链路收束到“解码成图”。

### 分镜 8 画面构图

- 采用单页问题拆解版式
- 中央放一张“看起来基本正确但局部有问题”的猫图示意
- 四周环绕 5 个误差来源标签

### 分镜 8 必备元素

- 左半区 5 个原因标签：
  - `seed 不同`
  - `prompt 约束不够强`
  - `guidance 过强或过弱`
  - `模型先验有限`
  - `步数 / 求解器选择不同`
- 中央异常示意标签：
  - `腿部异常`
  - `逆光不自然`
  - `动作不够像奔跑`

### 分镜 8 建议标注

- 左侧总结：`同一句 prompt，不一定每次都得到完全相同的画面`
- 中央异常示例标签：`腿部异常`、`逆光不自然`、`动作不够像奔跑`
- 页面收尾句：`整条流程到“解码成图”为止，结果是否稳定，取决于前面每一步条件约束和采样推进是否足够准确`

### 分镜 8 连线要求

- 5 个原因标签分别用短箭头指向中央异常图
- 不再出现额外流程顺序箭头

### 分镜 8 避免重复

- 不要再完整重画整条主链路
- 这里只做“影响结果的因素”，不再引入额外产品层内容

## 统一出图规范

### 统一术语

- 始终使用 `latent`，不要一页写“潜变量”、一页写“隐变量”、一页写“中间图”
- 始终使用 `文本编码器`，不要混写成“语言模型”或“翻译器”
- 始终使用 `去噪器`，必要时可在括号里补 `Transformer / DiT 类结构`
- 始终使用 `解码器 / VAE`，不要误写成“渲染器”

### 统一连线规则

- 实线箭头：真实流程推进
- 虚线箭头：辅助条件输入或可选分支
- 箭头方向必须从“输入/当前状态”指向“下一状态/下一模块”

### 统一风格规则

- 每页颜色尽量固定语义：
  - 蓝色：文本与条件
  - 紫色：latent 与采样
  - 橙色：图像与解码结果
  - 红色：风险、跑偏、异常项
- 每页主标题控制在 12 字以内
- 每页正文标注尽量短句化，适合直接上图，不写成长段解释

## 可直接给设计师的封底总结文案

`文生图不是“模型看完 prompt 一次性把图画出来”，而是先把 prompt 变成条件，再从随机噪声出发，在 latent 空间里经过多轮受条件约束的更新，最后解码成一张可见图片。`
