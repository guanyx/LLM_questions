# EAGLE 系列之三：EAGLE-3 进化之路——从特征回归到直接 Token 预测

最新的 **EAGLE-3** (NeurIPS 2025) 并不是简单的“修修补补”，而是在底层逻辑上进行了一次**范式转移**。它主要解决了前代版本的两个隐形瓶颈。

为了避免“听起来像换了个算法家族”，先给一个一眼看懂的对照：

- **EAGLE-1/2**：在 **Feature 空间**自回归（预测下一步 Feature，再经 LM Head 得到 Token）。
- **EAGLE-3**：在 **Token 空间**自回归（直接预测 Token 分布），不再做 Feature 回归。
- **信息来源升级**：从“只看输出端高层 Feature”升级到“融合低/中/高层特征 + TTT 多步训练模拟”。

---

## 1. 打破“模仿游戏”的枷锁 (Direct Token Prediction)

EAGLE-1/2 的核心逻辑是**“模仿”**：插件拼命去预测基座模型的 Feature。这意味着插件的上限被锁死在“逼近基座模型的输出端高层 Feature 分布”这件事上。
EAGLE-3 发现：**为什么非要模仿 Feature？我们的最终目标不是 Token 吗？**
于是，EAGLE-3 **放弃了 Feature 回归任务**，改为**直接预测 Token**。

- **解放约束**：插件不再需要小心翼翼地拟合那个高维 Feature 向量，它可以自由地学习任何能帮助预测 Token 的中间表示。
- **直觉**：就像学骑自行车，以前教练（EAGLE-1/2）要求你“膝盖弯曲角度必须和冠军一模一样”，现在教练（EAGLE-3）说“不管姿势怎样，只要骑得快就行”。

---

## 2. 全景视野 (Multi-layer Feature Fusion)

EAGLE-1/2 主要依赖**靠近输出端的高层 Feature**（例如 LM Head 前/倒数第二层）。但研究发现，这类高层特征往往过于关注“下一个词”，而丢失了“全局语义”。
EAGLE-3 引入了 **多层特征融合**：

- 它同时抽取大模型的 **浅层（Low）、中层（Mid）、深层（High）** 特征进行融合。
- **直觉**：这就像开车，以前只盯着前方 5 米（高层 Feature），现在同时用余光看路牌（Low）、看车流（Mid）、看导航（High）。视野更立体，预测当然更准。

---

## 3. 训练即实战 (Training-Time Test, TTT)

这是最硬核的改进。

- **痛点**：传统训练是“老师喂饭”（Teacher Forcing），输入永远是正确的 Ground Truth。但测试时，插件必须吃自己生成的（可能带有误差的）输出。这就导致了 **Train-Test Mismatch**（训练测试不匹配）。
- **EAGLE-3 方案**：在训练阶段，就故意让插件“自产自销”。它会把上一轮自己生成的（可能有瑕疵的）预测结果喂给下一轮。
- **效果**：插件在训练营里就习惯了纠错，产生了抗体。到了战场上（推理阶段），即使第一步走偏了一点点，它也能马上调整回来，而不是像以前那样一步错步步错。

---

## 4. 核心差异：扔掉“特征拐杖”

如果你读到这里觉得“EAGLE-3 和 EAGLE-1/2 像两种完全不同的物种”，通常是把两条变化轴混在一起了：

- **插件长什么样（形态）**：EAGLE-1/2/3 都是“很小的插件 + 复用基座模型 LM Head”这一套路，核心仍然可以是 **1 层 Transformer Decoder Layer**。
- **插件学什么（训练信号与状态变量）**：EAGLE-1/2 把“像不像基座模型的 Feature”当成硬指标；EAGLE-3 把这个指标拿掉，直接用“Token 对不对”驱动学习，并用 TTT 让训练时就暴露在真实推演条件下。

因此它看起来“像换了物种”，本质不是层数变了，而是**监督目标**和**自回归时携带的状态**变了。

把差异拆开看会更清楚：

1.  **EAGLE-1/2 (带着镣铐跳舞)**：

    - **输入**：$Input_t = Proj([\text{Embedding}(Token_{t-1}) ; \mathbf{Feature}_{t-1}])$
    - **痛点**：注意后面这个 $\mathbf{Feature}_{t-1}$。这不是“插件内部的隐状态”，而是插件**上一轮显式预测出来、并且被训练去逼近大模型某一层隐藏状态分布**的特征向量。
    - **后果**：插件每一轮都必须产出一个“像大模型”的特征喂给下一轮。这就好比一个实习生（插件）在写文章，他不仅要写出下一个字（Token），还必须每一秒都向导师汇报：“我现在脑子里的神经元活动是这样的（Feature），请检查对不对”。这极大地限制了实习生的发挥。

2.  **EAGLE-3 (扔掉拐杖奔跑)**：
    - **输入**：基座模型提供的**多层融合特征**（更丰富的上下文信息） + 插件已经草拟出的 Token 序列（TTT 让训练时就学会“吃自己的输出”）。
    - **推演方式**：插件在 **Token 层面**做自回归，直接输出下一步的 Token 分布；它当然仍然会在内部形成连续的隐藏状态，但这些隐藏状态**不再被要求长得像基座模型的某层 Feature**。
    - **解放**：关键不在于“它不看 Feature”（它依然利用基座模型的特征信号），而在于它**不再把 Feature 回归当成必须完成的硬作业**——目标只剩下把草稿 Token 预测准、让通过验证的比例更高。
    - **比喻**：以前导师要求你“写字的握笔姿势必须一模一样”（Feature 回归），现在导师只看“字写得对不对”（直接 Token 预测），并且训练时就让你在纸上连续写一整行（TTT 的多步模拟），所以到了真写作时更稳。

**战绩**：凭借这三板斧，EAGLE-3 在 Llama-3-8B 上实现了 **5x - 6x** 的加速比，且随着训练数据量的增加，性能呈现出持续上升的 Scaling Law 曲线（而旧版算法早就饱和了）。
