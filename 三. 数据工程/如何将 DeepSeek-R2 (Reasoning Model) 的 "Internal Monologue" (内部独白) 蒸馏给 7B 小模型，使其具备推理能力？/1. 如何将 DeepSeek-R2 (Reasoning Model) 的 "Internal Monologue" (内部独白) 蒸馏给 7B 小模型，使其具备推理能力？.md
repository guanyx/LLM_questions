# 如何将 DeepSeek-R2 (Reasoning Model) 的 "Internal Monologue" (内部独白) 蒸馏给 7B 小模型？

> **说明**：DeepSeek-R1 是目前公开的最强开源推理模型，本文基于 DeepSeek-R1 的技术报告和蒸馏方案编写（R2 为其后续迭代版本，核心蒸馏逻辑一脉相承）。我们将深入探讨如何通过“知识蒸馏”将大模型的强大推理能力（Internal Monologue / Chain of Thought）迁移到 7B 级别的小模型上。

## 1. 核心概念：什么是“内部独白”蒸馏？

传统的知识蒸馏（Knowledge Distillation）通常是让小模型学习大模型的 **"Final Answer"（最终答案）** 或 **"Logits"（概率分布）**。
但在推理任务（数学、代码、逻辑谜题）中，仅仅知道答案是不够的。

**DeepSeek-R1/R2 的核心突破在于：**
它不仅输出答案，还显式地输出了 **思维链（Chain of Thought, CoT）**，即 `<think>...</think>` 标签包裹的内容。

**蒸馏“内部独白”的本质：**
不仅仅是教小模型“答案是什么”，而是教它**“大模型是如何一步步思考出来的”**。这就像老师教学生，不是直接给答案，而是把解题草稿纸（Internal Monologue）展示给学生看，让学生模仿解题步骤。

---

## 2. 蒸馏全流程指南 (The Pipeline)

将 DeepSeek-R2 的能力蒸馏给 7B 小模型，主要分为以下三个关键步骤：

### 第一步：构建推理数据集 (Data Generation)

你需要利用强大的教师模型（DeepSeek-R1 或 R2）生成包含完整思维过程的数据。

1.  **Prompt 设计**：
    强制模型输出思维过程。DeepSeek 系列通常自带此能力，但你可以通过 System Prompt 强化：

    ```text
    Please process the user's request by first thinking explicitly inside <think> tags, and then providing the final answer.
    ```

2.  **生成数据 (Inference)**：
    让 Teacher Model 针对大量问题（如 Math, Code, Logic）生成回答。

    - **关键点**：保留 `<think>...</think>` 部分。不要清洗掉！这是蒸馏的灵魂。

3.  **质量控制 (Filtering / Rejection Sampling)**：
    这是最重要的一步。大模型也会胡说八道。你需要验证生成的**答案**是否正确。
    - **对于数学题**：比较最终数值答案。
    - **对于代码题**：运行 Unit Test。
    - **策略**：对于同一个问题，让 Teacher 生成 10 次，只保留那些**答案正确**且**格式规范**的样本。DeepSeek 团队在 R1 中使用了这种 **Rejection Sampling（拒绝采样）** 技术。

### 第二步：数据格式化 (Formatting)

将清洗后的数据整理成 SFT（Supervised Fine-Tuning）所需的格式。通常如下：

```json
{
  "instruction": "Solve the following equation: 2x + 5 = 15",
  "input": "",
  "output": "<think>\nTo solve for x, I need to isolate it.\n1. Subtract 5 from both sides: 2x = 10.\n2. Divide by 2: x = 5.\nIt seems correct.\n</think>\nThe answer is 5."
}
```

**注意**：

- `<think>` 标签是特殊的 Token 或格式，让小模型学会“先思考，后回答”的模式。
- DeepSeek 的蒸馏实验表明，**直接使用 SFT**（监督微调）在高质量的长思维链数据上，就能让 7B 模型获得惊人的推理能力，甚至不需要复杂的强化学习（RL）。

### 第三步：监督微调 (Supervised Fine-Tuning, SFT)

使用你选择的 7B 基座模型（如 Qwen2.5-7B, Llama-3-8B, 或 DeepSeek-V3-Base），在上述数据集上进行微调。

- **工具推荐**：`LLaMA-Factory`, `Unsloth`, `Axolotl`.
- **Loss 计算**：
  - 模型在训练时，会学习预测 `<think>` 里面的每一个字符。
  - 这意味着模型被迫学习大模型的“思考逻辑”、“自我纠错”和“试错过程”。
- **Context Length（上下文长度）**：
  - 推理模型的 `<think>` 部分可能非常长（数千 tokens）。确保你的 7B 模型训练时 Context Window 足够大（至少 4k-8k，推荐 16k+）。

---

## 3. 进阶：为什么仅仅 SFT 就够了？

在 DeepSeek-R1 的论文中，一个反直觉的结论是：
**对于小模型（Student），只要有足够高质量的 Teacher Reasoning Data，简单的 SFT 就能达到极高的效果。**

大模型（Teacher）需要使用 **RL (Reinforcement Learning, specifically GRPO)** 来“顿悟”出推理能力。但一旦大模型学会了，它生成的 `<think>` 数据就包含了所有的智慧。小模型只需要通过 SFT “背诵”和“模仿”这些思维路径，就能获得泛化能力。

### 蒸馏 vs. 强化学习 (RL)

- **DeepSeek-R1 (Teacher)**: 使用了大规模 RL (GRPO) 进行自我进化，涌现出长思维链。
- **DeepSeek-R1-Distill (Student)**: 主要是 SFT。如果资源允许，可以在 SFT 之后再加一轮 RL，但性价比最高的步骤是 SFT。

## 4. 实战：如何自己动手做？

如果你想尝试将 DeepSeek 的能力蒸馏给自己的 7B 模型：

1.  **准备种子数据**：收集 50k - 100k 条高质量的 Math/Code 问题（如 GSM8K, MATH, MBPP 数据集）。
2.  **调用 API 生成思维链**：
    使用 DeepSeek API (model=`deepseek-reasoner`)，对每个问题获取 `reasoning_content`。
    _(注意：DeepSeek API 将 `reasoning_content` 和 `content` 分开返回，你需要拼接)_
3.  **拼接数据**：
    ```python
    target_output = "<think>" + reasoning_content + "</think>\n" + final_content
    ```
4.  **微调 (Fine-tune)**：
    使用 `Unsloth` 对 `Llama-3-8B-Instruct` 或 `Qwen2.5-7B` 进行微调。
    - 设置 `learning_rate` 约为 2e-5。
    - 使用 `packing` (sequence packing) 提高训练效率。
5.  **评估**：
    测试模型是否会在回答困难问题前自动输出 `<think>`，以及推理逻辑是否连贯。

## 5. 总结

将 DeepSeek-R2/R1 的 "Internal Monologue" 蒸馏给 7B 模型的秘诀在于：**“不仅学结果，更要学过程”**。

通过保留并训练 `<think>` 标签内的思维流，7B 模型实际上是在学习一种**“通用的解决问题算法”**，而不仅仅是文本生成的概率分布。这就是为什么蒸馏后的小模型能展现出超越同尺寸模型的数学和逻辑能力的原因。
