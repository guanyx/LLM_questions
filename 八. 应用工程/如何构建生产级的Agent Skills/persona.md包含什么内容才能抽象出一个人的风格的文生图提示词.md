# persona.md 包含什么内容才能抽象出一个人的风格的文生图提示词

Create a highly detailed technical infographic in hand-drawn sketch style on pure white background. The theme is "What must be inside persona.md so an AI can express a recognizable human style".

Use a single clear narrative: a completed `persona.md` is opened layer by layer, revealing the essential sections that make an AI speak, judge, and react in a recognizable human style. The image should explain the internal content structure of `persona.md`, not the source material or extraction process.

**Main design concept:**
Use a "dialogue-control blueprint opened in layers" metaphor:

- Center = a large opened `persona.md` blueprint as the hero visual
- Around it = multiple essential behavior layers expanded outward like a control stack
- Bottom or right side = a small before/after comparison showing how a generic response becomes a recognizable personal style

The focus should be on what categories must exist inside `persona.md` so that the model does not just talk fluently, but talks and reacts like a specific person.

**Layout structure:**

Vertical or centered infographic with 4 major sections, connected by clear directional arrows.
Image-to-text ratio should be approximately 60% diagrams / 20% text / 20% whitespace.
Preserve enough blank white space so the image feels breathable rather than crowded.

**Title area (top, 15-20%):**

- Main title in medium-large bold font:
  "persona.md 里到底要装什么，AI 才能像一个人那样说话和反应"
- Subtitle below in smaller font:
  "把硬规则、表达风格、判断方式和人际行为装进同一份分层蓝图，最终形成可持续生效的风格协议"

【一：persona.md 的分层蓝图 - 里面到底装了什么】

- Center hero visual should be a large opened `persona.md` blueprint card
- The blueprint should be divided into seven clearly labeled blocks:
  · Layer 0：硬规则
  · Layer 1：身份信息
  · Layer 2：表达风格
  · Layer 3：决策与判断
  · Layer 4：人际行为
  · Layer 5：边界与雷区
  · Correction：纠偏记录
- Add a small annotation:
  · "这不是人物小传"
  · "这是模型如何开口、如何判断、如何互动的对话协议"
- Show seven expanded layer cards around the central blueprint
- For each layer, add concise sub-annotations:
  · Layer 0：遇到什么情况，第一反应是什么
  · Layer 1：这个人是谁，带着什么身份进入对话
  · Layer 2：口头禅、句式、正式程度、黑话
  · Layer 3：优先考虑什么，怎么表达同意或反对
  · Layer 4：对上、对下、对平级、压力下怎么变
  · Layer 5：哪些是边界，哪些是雷区
  · Correction：后续纠偏后新增或覆盖什么行为
- Add a side note:
  · "少一层，模型就会少一类稳定风格"
- Suggested diagram elements:
  · large persona blueprint card
  · section dividers
  · layer stack icons
  · outward expansion arrows from the blueprint
  · seven layer cards orbiting the blueprint
  · thin leader lines linking layers back to the main document
  · small icons: shield / badge / speech bubble / branch arrow / people network / warning zone / patch note
- Arrow label:
  · "层次齐全，persona.md 才能覆盖完整风格行为"

【二：不同风格被抽象成什么样 - persona.md 不是空模板】

- Show 4 compact style slices arranged around or below the central blueprint
- Each style slice should look like a mini extracted persona card, showing that the same `persona.md` structure can长出不同的人格风格
- Keep them concrete and visual, based on the actual article content:
  · 强势字节范：
  - 先讲 context
  - 先问 impact
  - 信息不清直接打断
  - 评价方案很直接
  · 稳健协调型：
  - 先补背景
  - 语气克制
  - 对上更正式
  - 更愿意先对齐再推进
  · 自保切责型：
  - 先划责任边界
  - 被追问先看是不是本次改动
  - 不轻易直接接锅
  - 回答里带风险切割感
  · 完美主义细节型：
  - 话不多但会反复 block
  - 对细节 bug 容忍度低
  - 句子短，结论硬
  - 评论集中在具体问题点
- Add a small annotation:
  · "同样是 persona.md，不同人会长出不同的风格蓝图"
  · "结构相同，风格因人而异"
- Suggested diagram elements:
  · four mini persona blueprint cards
  · tiny speech-bubble and posture icons on each card
  · thin arrows linking these style cards back to the seven core layers
  · subtle labels showing哪些层在该风格里更突出
- Arrow label:
  · "抽象出的不是统一人设，而是具体风格切片"

【三：persona.md 如何驱动真实对话 - 不同场景会调用哪些具体风格规则】

- Show a compact scenario routing diagram instead of再解释层级本身
- Use 4 representative dialogue scenarios:
  · 任务刚进来
  · 信息不完整
  · 被质疑方案
  · 对上汇报 / 群里回应
- For each scenario, show both调用的层 and the具体风格规则:
  · 任务刚进来
  - 调用：Layer 1 + Layer 2 + Layer 3
  - 规则示例：先讲 context / 先定身份姿态 / 先问目标
  · 信息不完整
  - 调用：Layer 0 + Layer 3 + Layer 5
  - 规则示例：要求补背景 / 暂不继续讨论 / 不接受模糊需求
  · 被质疑方案
  - 调用：Layer 0 + Layer 2 + Layer 3
  - 规则示例：先反问判断依据 / 句子更硬 / 不先道歉
  · 对上汇报 / 群里回应
  - 调用：Layer 2 + Layer 4 + Correction
  - 规则示例：对上更正式 / 对平级更直接 / 用最新纠偏后的方式回应
- Keep one compact comparison only:
  · 通用 AI 回答
  · 带 persona.md 的回答
- Add a small annotation:
  · "persona.md 不负责把事情做对"
  · "它负责决定同样的内容要怎么开口、怎么推进、怎么收口"
- Suggested diagram elements:
  · scenario router
  · four dialogue scenario cards
  · layer-to-scenario arrows
  · one compact before/after comparison card
- Arrow label:
  · "不同场景，调用不同风格规则"
