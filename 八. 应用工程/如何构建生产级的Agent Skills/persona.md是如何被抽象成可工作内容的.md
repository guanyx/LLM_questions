# persona.md 是如何被抽象成可工作内容的

## 一、先说结论

`persona.md` 不是“给同事写人设小传”，而是把一个人在工作场景中的说话方式、判断方式和人际反应，压缩成一组在对话时可持续生效的行为规则。

如果说 `work.md` 解决的是“这件事怎么做”，那么 `persona.md` 解决的是“这个人会用什么态度来接这件事，并且会怎么说”。它之所以能变成可工作的内容，是因为项目没有停留在性格标签，而是把标签和原始语料一起改写成分层控制规则。

## 二、persona.md 抽象的输入是什么

这一层有两个主要来源：

- 用户手工输入的先验标签，比如 MBTI、企业文化、主观印象、个性标签
- 原材料中能反映表达与行为的证据，比如聊天记录、邮件、评审回复、会议纪要

这里有一个很重要的设计：手工标签优先级高于文件分析。

也就是说，项目默认相信“你认识这个人”，然后再用文本证据去补细节、做校准，而不是完全交给模型自由归纳。

## 三、第一步：先把标签翻译成硬规则

`persona.md` 最核心的并不是风格模仿，而是 Layer 0。

这一层会把“甩锅高手”“字节范”“完美主义”“向上管理专家”之类的标签，翻译成明确的行为规则。例如：

- 遇到问题先找外部原因，不主动认错
- 没有讲清 context 时先打断，要求补背景
- 评价方案先问 impact，答不上来就不继续讨论
- 被安排不想做的事时，优先转包而不是正面拒绝

这一步很关键，因为它把抽象人格词变成了模型能执行的条件动作对。模型不需要理解“字节范”的文化史，只需要知道“在什么场景下应该怎么反应”。

## 四、第二步：从语料里抽表达风格

接下来，项目会从消息、邮件、评审文本里抽取更细的表达特征，包括：

- 高频词和口头禅
- 内部黑话和固定搭配
- 句子长短
- 结论前置还是先铺垫
- 是否喜欢列点
- emoji、标点和正式程度

这一步的目标不是做语言学分析，而是把“像不像这个人”最容易被感知到的外层特征提炼出来。因为对用户来说，像不像往往先体现在语气、节奏、措辞，而不是深层价值观。

## 五、第三步：抽决策模式，而不是只抽语气

如果只有语言风格，`persona.md` 会变成一个会学口癖的聊天壳子。所以项目继续往下抽一层：决策模式。

这一层会关注：

- 他遇到分歧时优先考虑什么，是效率、流程、数据、人情还是政治
- 什么情况会主动推进
- 什么情况会拖延、回避、甩给别人
- 他怎么表达不同意
- 他被质疑时会解释、反问、认错还是转移

这一步决定了模型不只是“像他说话”，还会“像他做判断”。从 Agent 工程角度看，这是把对话代理从风格模仿，推进到行为模拟。

## 六、第四步：抽人际关系模式

再往下一层，项目会继续抽这个人在不同关系里的行为切换：

- 对上级怎么汇报、怎么邀功、出了问题怎么说
- 对下级怎么分配任务、愿不愿意辅导、别人犯错时怎么处理
- 对平级怎么协作、怎么划边界、群里是活跃还是潜水
- 压力大、被催、被怀疑、被甩锅时会发生什么变化

这一层很像给代理补上“社会接口”。因为真实的人并不是用同一种方式对所有人说话，很多“像本人”的感觉，恰恰来自这些关系化差异。

## 七、第五步：做成分层 Persona，而不是一段描述

最后，前面抽出来的内容不会被写成一段自由散文，而是被放进多层结构里：

- Layer 0：硬规则
- Layer 1：身份信息
- Layer 2：表达风格
- Layer 3：决策与判断
- Layer 4：人际行为
- Layer 5：边界与雷区
- Correction：后续纠偏记录

这种分层设计非常重要，因为它等于给模型定义了优先级。

比如“他平时礼貌”这种风格，如果和“被质疑时先反问”这种硬规则冲突，模型应该优先遵守更高层的行为规则，而不是继续扮演礼貌语气。这样 `persona.md` 才不会变成彼此打架的人设拼盘。

## 八、第六步：让它在对话中可持续工作

`persona.md` 真正“可工作”的地方，不在于它写得像小说，而在于它最终会参与任务执行顺序：

- 先由 Persona 判断要用什么态度接任务
- 再由 Work Skill 负责把任务做完
- 输出时始终保持 Persona 的表达风格

这意味着 `persona.md` 并不是展示层装饰，而是整个代理链路的前置控制器。它决定模型是直接答、先追问、先质疑、先甩锅，还是先拉齐背景。

## 九、第七步：通过纠正机制继续拟合

人格抽象很难第一次就准，所以项目专门留了 Correction 层。

当用户说“他不会这样”“他应该会先问 impact”“他被质疑时不会解释”，系统会把这些话转成标准化纠正规则，追加到对应层里。

这相当于给 `persona.md` 增加了一个在线微调接口，只不过调的不是模型参数，而是行为规则本身。

## 十、为什么它最后会变成“可工作的”

因为它不是在写人物介绍，而是在写一套对话控制协议。整个过程做了三件事：

- 把主观标签翻译成硬行为规则
- 把语料中的表达、判断、人际模式抽成稳定特征
- 把这些特征装进有优先级的多层 Persona 结构

这样一来，模型拿到的就不是“某人性格比较强势”这种模糊描述，而是“在什么情境下用什么口气、做什么判断、给出什么反应”的具体约束。

## 十一、一句话总结

`persona.md` 的抽象过程，本质上是把“这个人过去如何表达与反应”的证据，压缩成“模型现在如何开口、如何判断、如何互动”的分层规则系统。
