# Harness Engineering 是什么：第 1 张封面图文生图脚本

Create an eye-catching technical cover infographic for social media (Xiaohongshu) in hand-drawn sketch style on pure white background. The theme is "Harness Engineering 是什么". The image should immediately communicate that Harness Engineering is the engineering system wrapped around a model core: it constrains, guides, stabilizes, observes, and routes model capability into a deliverable runtime behavior.

**Design Concept:**

Use a "harness / reins / control straps around a model core" visual metaphor, not a horse illustration. The center should be a glowing, unstable, high-capability model core. Around it, build an outer ring of semi-transparent engineered straps, control bands, and directional reins that hold, guide, and stabilize the model. These outer engineering bands should visibly represent tools, permissions, context, workflow, memory, evals, and recovery. The feeling should be: the model is powerful, but only becomes usable when the surrounding engineering harness turns raw intelligence into controlled system behavior.

**Layout Structure:**

**Top Section (Title Area - 22%):**

- Main title in bold, large font: "Harness Engineering 是什么"
- Subtitle below: "模型外面那一圈把智能套住、导向、稳定下来的工程系统"
- Small lead sentence under subtitle:
  "真正能落地的 Agent，不只是模型本身，而是模型外面那套能约束、编排、观察、恢复并控成本的系统"
- Strong typography hierarchy, centered layout, clean technical-poster feel

**Central Section (Hero Visual - 58%):**

Create one dominant central composition with a strong "model core + outer harness system" relationship:

- Center object:
  - a luminous "模型核心" / "model core" sphere or reactor-like intelligence core
  - should feel energetic, slightly unstable, full of internal activity
  - use subtle particle noise, internal wave lines, or neural lattice texture to suggest powerful but uncontrolled reasoning
  - add a small internal label: "Model Core"

- Outer harness ring:
  - do NOT use a hard shell, cage, prison bars, or metal enclosure
  - instead, wrap the core with 7 semi-transparent harness straps / reins / control bands coming from different directions
  - straps should visibly tighten around and guide the core, but not feel violent
  - each strap should be structurally anchored to an external engineering module
  - each strap labeled in Chinese with compact English keyword:
    - 工具系统 / tools
    - 权限边界 / permissions
    - 上下文工程 / context
    - 工作流编排 / workflow
    - 记忆系统 / memory
    - 评测观测 / evals
    - 恢复机制 / recovery

- Engineering anchor modules:
  - place 7 compact module cards around the outer area, each corresponding to one harness strap
  - cards should not be too large; they are support structures that visually "pull and shape" the model core
  - each module card can include one tiny internal annotation:
    - tools: 文件 / 浏览器 / API
    - permissions: allow / block / approve
    - context: 检索 / 压缩 / 注入
    - workflow: planner / executor / reviewer
    - memory: 偏好 / 状态 / 经验
    - evals: trace / replay / logs
    - recovery: retry / fallback / route

- Guided output direction:
  - from the right front side of the harnessed core, create one clean, stable output stream moving toward a structured delivery panel
  - this stream should feel very different from the noisy core: cleaner, narrower, more controlled, more system-like
  - destination panel labeled:
    - 可交付运行结果
    - runtime system output
  - inside or near this panel, use tiny structured tags:
    - stable
    - bounded
    - observable
    - recoverable

- Visual contrast inside hero area:
  - left/rear side of the model core should still show some diffuse, noisy, unbounded energy
  - right/front side should show engineering-guided output
  - make it visually obvious that the harness does not "kill" intelligence; it channels it

- Structural guidance:
  - use thin blueprint leader lines to point from callouts to straps and modules
  - add subtle directional arrows from module cards to the core
  - show that the outer layer is not decorative; it is actively shaping the model

**Bottom Section (Reading Guide - 20%):**

Create a compact horizontal reading guide showing the major layers that this cover will expand in later slides:

1. "为什么突然重要"
2. "工具与权限"
3. "上下文与流程"
4. "记忆与评测"
5. "成本与恢复"

- Connect them with clean arrows showing progression
- Add one guide sentence:
  "后面 7 张，逐层拆开这套工程缰绳"
- Add a small footer label:
  "Harness Engineering / 01"

**Key Visual Metaphor:**

- The image must feel like "工程缰绳套住模型核心"
- Not horse riding, not a cowboy scene, not an animal anatomy scene
- The harness should look like engineered directional control bands around intelligence
- The center is raw intelligence
- The outer system is control, routing, boundaries, observation, and stabilization
- The final message should be:
  "智能不是直接交付，必须先被 Harness 成系统"
