# Hermes Agent对象拆解：第 13 张Plugin / MCP / External Memory文生图脚本

Create a highly detailed technical infographic for social media in hand-drawn sketch style on pure white background. The theme is "Hermes 为什么还能继续长能力？".

**Design Concept:**
Use a platform-expansion stack metaphor instead of concentric rings: the image should show a stable Hermes Core on the left, and three clearly differentiated extension lanes on the right. The key message is not simply “more integrations”. It is that Hermes is designed as a platform that can keep growing new organs through plugins, MCP servers, and external memory providers without rewriting the core.

**Layout Structure:**

**Top Section (Title Area - 20%):**

- Main title in bold, large font: "Hermes 为什么还能继续长能力？"
- Subtitle below: "核心负责稳定，扩展层负责让 Hermes 不断长出新器官"
- Small supporting sentence: "Hermes 不把能力全部写死在核心里，而是通过插件、MCP 和外部记忆 provider 形成持续生长的扩展层"
- Typography should feel like a systems-platform poster, not a marketplace ad

**Central Section (Hero Diagram - 55%):**
Create a left-to-right platform-expansion architecture diagram with a stable core and three extension lanes:

- On the far left place one strong, compact core labeled "Hermes Core"
  - inside show 4 tiny labels:
    - 核心运行时
    - 主循环
    - 工具系统
    - 记忆协调
  - make the core look dense, protected, and intentionally stable

- From Hermes Core, draw one thick horizontal trunk line toward the right labeled:
  - "扩展接口层"
  - annotation: "核心不必频繁改写，扩展从统一边界接入"

- On the right side, split the trunk line into 3 stacked extension lanes from top to bottom:

- Lane 1 - Plugins
  - label: "Plugins"
  - visual cue: modular blocks docked onto a capability rail
  - add example cards on this lane:
    - GitHub
    - DB
    - Browser Stack
  - annotation: "把外部系统能力接成可调用模块"

- Lane 2 - MCP Servers
  - label: "MCP Servers"
  - visual cue: protocol-linked server boxes on a clean horizontal bridge
  - add note:
    - "通过协议接外部能力，而不是把所有工具写死在仓库里"
  - add one or two generic server nodes to imply extensibility

- Lane 3 - External Memory Providers
  - label: "External Memory Providers"
  - visual cue: memory provider cartridges / storage nodes on a separate lane
  - add example cards:
    - Honcho
    - Hindsight
    - Mem0
    - RetainDB
    - Supermemory
  - annotation: "记忆层也可以外挂，不必只依赖内建实现"

- Make the three lanes visibly different in function:
  - Plugins should feel like functional modules
  - MCP should feel like protocol/bridge access
  - External Memory should feel like pluggable cognition/storage layers

- Add one top-side growth arrow moving from left to right across the three lanes:
  - label: "能力持续生长"
  - annotation: "新增能力主要长在扩展层，不必频繁改写核心"

- Add one lower-side stabilizing note attached to Hermes Core:
  - label: "核心保持稳定"
  - annotation: "扩展再多，核心仍然负责运行时一致性"

- Add one green design-strength callout:
  - label: "设计亮点"
  - annotation: "MCP、插件、外部记忆都做成可插拔层，Hermes 因此不只是应用，也开始像平台"

  - label: "设计代价"
  - annotation: "扩展层越强，兼容性、依赖、可信边界、生态治理就越复杂"

- Add one bold architectural judgment note near the lower center:
  - "这一层表达的不是“外挂更多”，而是“Hermes 已经开始平台化”"

- Add footer label:
  "Hermes Agent 对象拆解 / 13"

**What Must Be Clear in the Diagram:**

- Hermes Core should feel stable and intentionally compact compared with the extension lanes
- The three extension lanes represent different semantics, not just three categories of integrations
- Growth happens from left to right through clear extension paths
- External memory providers should be presented as real architecture participants, not as optional footnotes
- The image must visually communicate this judgment:
  "Hermes 不只是产品，也开始像平台"
