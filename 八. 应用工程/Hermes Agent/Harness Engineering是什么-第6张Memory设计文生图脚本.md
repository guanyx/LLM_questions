# Harness Engineering 是什么：第 6 张“Memory 设计”文生图脚本

Create a highly detailed technical infographic for social media in hand-drawn sketch style on pure white background. The theme is "记忆不是越多越好，而是越有边界越好". The image should make it immediately clear that mature agent memory is not a giant pile of chat history, but a structured state system with separated layers and different retention purposes. This should feel like a "front-facing archive cabinet cutaway / memory storage cross-section" diagram, not another left-to-right rectification flow.

**Design Concept:**

Use a "记忆档案柜剖面 / archive storage anatomy" metaphor:

- top section = incoming raw memory inbox
- center spine = summarize / isolate / store write bus
- lower main body = layered memory cabinet
- side retrieval window = how future tasks read memory back

The key visual judgment is:

**Harness 里的记忆，真正要保存的不是过去说过的话，而是未来还要用的状态。**

The image should make viewers feel that memory is not about keeping everything forever. Memory is about deciding what should be retained, how it should be categorized, and who should read it later. The reading order must be obvious at first glance:

- top = incoming memory materials
- middle = write and classification logic
- bottom = stored memory layers
- side = future retrieval

**Layout Structure:**

**Top Section (Title Area - 20%):**

- Main title in bold, large font: "记忆不是越多越好，而是越有边界越好"
- Subtitle below: "真正成熟的记忆，不是堆聊天记录，而是分层保存可复用状态"
- Small supporting sentence:
  "长期任务不是不能记，而是不能什么都记、一起记、永远记。真正值钱的是能在未来任务里再次被正确取出的那部分状态"
- Typography should feel like a technical editorial blueprint, not a soft consumer AI memory poster

**Central Section (Hero Diagram - 58%):**

Create one dominant front-facing archive cabinet cutaway composition with four explicit zones:

- Top zone - 待处理记忆入口:
  - build an incoming "memory inbox" tray at the top of the composition
  - represent mixed memory materials dropping into this tray as overlapping slips, chat strips, task notes, and sticky labels
  - include incoming materials such as:
    - 聊天历史
    - 任务状态
    - 用户偏好
    - 旧结论
    - 临时上下文
    - 已过期事实
    - 私密信息
    - 可复用经验
  - these materials should feel mixed, unsorted, and not yet trusted
  - add tiny warning tags around the tray:
    - 过时
    - 混用
    - 重复
    - 泄漏风险
    - token 膨胀
  - add one small top label:
    - "Memory Inbox"

- Center vertical spine - Memory Write Bus:
  - this is the key engineering layer of the image
  - place one central write spine descending from the top inbox into the cabinet below
  - split the spine into 3 processing checkpoints:
    - summarize
    - isolate
    - store
  - each checkpoint should have compact Chinese explanations:
    - summarize: 把长历史压成短而稳定的结论
    - isolate: 不同类别、不同角色、不同敏感级别分开
    - store: 按用途写入不同抽屉，而不是全部堆进一个池子
  - show some fragments being compressed, some being routed, and some being rejected or faded out
  - add one note near the spine:
    - "记忆的关键不是都留下，而是写进正确的位置"

- Lower main body - 分层记忆柜:
  - build a large, front-facing memory cabinet occupying the lower half of the page
  - split it into 4 drawers or compartments with strong visible separators:
    - user preferences
    - task state
    - reusable experience
    - long-term facts
  - each drawer should have one short Chinese explanation:
    - user preferences: 长期偏好 / 风格 / 习惯
    - task state: 当前任务进度 / 中间状态 / 下一步
    - reusable experience: 可复用做法 / 成功路径 / 经验总结
    - long-term facts: 稳定事实 / 长期知识 / 用户资料
  - the cabinet should look precise, labeled, and retrievable
  - drawer boundaries must be visually strong so "memory categories" feel real

- Right-side retrieval window:
  - add one smaller side panel on the right, separate from the cabinet body
  - label it:
    - "下一次任务读取"
  - show one or two retrieval arrows coming from specific drawers toward this side panel
  - annotate the read logic with tiny notes:
    - 只读相关抽屉
    - 不必重放全部历史
    - 未来任务读的是状态，不是整段聊天

- Boundary emphasis:
  - add one small contrast note near the top inbox:
    - "全部一起记，看起来完整，实际上最容易污染未来任务"
  - add one small note near the cabinet drawers:
    - "不同记忆，不该混在同一个池子里"

- Benefit contrast:
  - add one orange callout near the top or upper-left:
    - "低级记忆：全都记住 -> irrelevant history / stale facts / privacy mixing / token inflation"
  - add one teal callout near the cabinet body:
    - "成熟记忆：分层、分类、隔离 -> 更稳、更省、更可复用"

**Bottom Section (Interpretation Strip - 22%):**

Create a bottom horizontal interpretation strip with 4 compact stages:

1. "进入收纳箱"
   - tiny explanation: "各种记忆材料先汇总"
2. "写入总线"
   - tiny explanation: "压缩、隔离、决定写哪层"
3. "分层抽屉"
   - tiny explanation: "不同用途，不同存储位"
4. "未来读取"
   - tiny explanation: "下次任务只取相关部分"

- Connect the four stages with arrows
- Add one bottom conclusion sentence:
  "Memory 设计真正解决的，不是‘记不记得住’，而是‘未来还能不能准确地读回来’"
- Add footer label:
  "Harness Engineering / 06"

**What Must Be Clear in the Diagram:**

- The top inbox must feel like a bad memory strategy, not simply "more memory"
- The central write spine must visibly do transformation and routing work
- The lower cabinet must feel like a structured storage system, not a pile with nicer labels
- The image must not imply that chat history alone equals memory
- The viewer should immediately understand:
  - memory is state, not only conversation
  - memory quality depends on boundaries
  - future retrieval matters more than raw retention

**Suitable Diagram Types to Combine:**

- Front-facing archive cabinet cutaway
- Incoming memory inbox tray
- Central write bus / routing spine
- Layered memory drawers
- Side retrieval window
- Bottom interpretation strip

**Visual Style:**

- Hand-drawn technical sketch aesthetic
- Professional engineering documentation style
- White background with strong top-to-bottom reading order and one side retrieval window
- Dense annotations but strong drawer/category boundaries
- More like a structured runtime-memory archive anatomy page than a human brain metaphor poster
- Composition ratio:
  - 55% detailed schematic illustration
  - 20% labels and technical callouts
  - 25% whitespace

**Color Palette:**

- Deep navy (#2C3E50) for outlines, titles, and memory-cabinet frames
- Bright blue (#3498DB) for processing arrows, archival flow, and retrieval lines
- Teal (#16A085) for structured memory compartments and reusable-state benefits
- Orange (#E67E22) for stale memory, mixed privacy, and overflow warnings
- Light gray for helper lines, old-history fragments, and faded discarded items
- Pure white background

**Typography:**

- All annotations in simplified Chinese
- Keep only these technical terms in English where appropriate: Memory Dump, summarize, isolate, store, user preferences, task state, reusable experience, long-term facts
- Title should be bold and large
- Secondary labels should be compact but readable on mobile

**Overall Feel:**

- Clear and highly readable at a glance
- Technical but not abstract
- Makes the difference between "everything remembered" and "structured reusable memory" visually unforgettable
- Immediately communicates that memory design is about boundaries, categories, and future retrieval
- Suitable for Xiaohongshu-style knowledge carousel sixth slide

Style: Detailed technical hand-drawn infographic, front-facing memory archive cabinet cutaway, incoming memory inbox routed through write bus into layered reusable drawers, pure white background
