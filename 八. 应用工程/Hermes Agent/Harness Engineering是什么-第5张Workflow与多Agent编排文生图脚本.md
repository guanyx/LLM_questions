# Harness Engineering 是什么：第 5 张“Workflow 与多 Agent 编排”文生图脚本

Create a highly detailed technical infographic for social media in hand-drawn sketch style on pure white background. The theme is "真正该设计的不是一个万能 Agent，而是任务怎么被拆开". The image should make it immediately clear that multi-agent is not the goal; workflow design and role separation are the goal. This should feel like a balanced comparison board: left side = one overloaded generalist agent, right side = a clean staged workflow with separated responsibilities.

**Design Concept:**

Use a "单脑过载 vs 分工流水线" metaphor:

- left side = one generalist agent carrying planning, retrieval, execution, and review all at once
- right side = a structured workflow where different steps own different responsibilities

The key visual judgment is:

**Workflow 设计的核心，不是让更多 Agent 同时上场，而是让每一步只承担自己该承担的职责。**

The image should make viewers feel that the real problem is not "do I have one agent or three agents", but "did I separate planning, execution, and checking clearly enough?" The reading order must be obvious at first glance: left = overload, center = workflow split, right = clearer execution chain.

**Layout Structure:**

**Top Section (Title Area - 20%):**

- Main title in bold, large font: "真正该设计的不是“一个万能 Agent”，而是任务怎么被拆开"
- Subtitle below: "多 Agent 不是目的，职责分离才是目的"
- Small supporting sentence:
  "复杂任务本来就不是一步完成。真正让系统变稳的，不是代理数量变多，而是每一步终于只做自己该做的事"
- Typography should feel like a technical editorial blueprint, not a futuristic multi-agent poster

**Central Section (Hero Diagram - 58%):**

Create a balanced left-to-right comparison board with three clear regions:

- Left region - 单脑过载区:
  - place one dominant "万能 Agent" core or brain-like runtime box
  - the box should be visibly overloaded with too many incoming and outgoing arrows
  - feed 4 responsibility streams into the same single core:
    - planning
    - retrieval
    - execution
    - review
  - each responsibility stream should have its own tiny visual cue:
    - planning: 任务拆解 / 下一步判断
    - retrieval: 查资料 / 找上下文
    - execution: 调工具 / 改状态
    - review: 检查结果 / 回看风险
  - inside the overloaded core, add compact labels:
    - 目标混在一起
    - 状态挤在一起
    - 错误难定位
  - add a few small warning tags around the core:
    - 干扰
    - 串线
    - 忘目标
    - 难评测
  - add one bottom judgment note:
    - "所有事都塞给一个 Agent，看起来省事，实际上最容易失控"

- Center region - Workflow 拆分带:
  - create a narrow but clear transition strip between left and right
  - label it:
    - Workflow Split
  - show the shift from:
    - all-in-one
    - to role-separated
  - include 3 compact shift reasons:
    - 降低互相干扰
    - 状态边界更清楚
    - 错误更容易定位
  - add one small note:
    - "多 Agent 只是实现形式，真正值钱的是流程拆分"

- Right region - 分工流水线区:
  - create one staged workflow with 3 main role blocks arranged left-to-right or top-to-bottom within the right panel:
    - planner
    - executor
    - reviewer
  - each block should own a clear input and output
  - each role block should contain compact Chinese explanations:
    - planner: 拆任务 / 定下一步 / 决定用什么路径
    - executor: 调工具 / 执行动作 / 回填结果
    - reviewer: 对照目标 / 检查风险 / 决定是否收敛
  - connect them with clean arrows:
    - task -> planner -> executor -> reviewer
  - add one thin feedback loop from reviewer back to planner or executor:
    - 需要重做时再回流
  - this right-side chain should feel clean, inspectable, and structured

- State clarity comparison:
  - on the left, add a small note:
    - "状态都堆在一个脑子里"
  - on the right, add a small note:
    - "每一步只持有自己需要的状态"

- Evaluation clarity comparison:
  - add one orange callout on the left:
    - "失败时，你很难知道错在规划、检索、执行还是审查"
  - add one teal callout on the right:
    - "角色分离以后，流程更容易回看，也更容易做评测"

**Bottom Section (Interpretation Strip - 22%):**

Create a bottom horizontal interpretation strip with 4 compact stages:

1. "任务复杂化"
   - tiny explanation: "不再是一轮做完"
2. "单脑过载"
   - tiny explanation: "规划、执行、审查混在一起"
3. "流程拆分"
   - tiny explanation: "把职责切开"
4. "稳定编排"
   - tiny explanation: "更清楚、更可控、更好评测"

- Connect the four stages with arrows
- Add one bottom conclusion sentence:
  "多 Agent 的真正价值，不在数量，而在职责分离以后，系统终于开始像流程而不是像即兴发挥"
- Add footer label:
  "Harness Engineering / 05"

**What Must Be Clear in the Diagram:**

- Left side is not "single agent is bad" in an absolute sense
- The real problem is role overload and state mixing
- Right side should not feel like random agent proliferation
- The value of the right side is:
  - clearer responsibilities
  - cleaner state boundaries
  - easier evaluation and debugging
- The viewer should immediately understand:
  - multi-agent is not the point
  - workflow decomposition is the point

**Suitable Diagram Types to Combine:**

- Balanced comparison board
- Overloaded single-core diagram
- Workflow split transition strip
- Structured role pipeline
- Feedback loop arrow
- Bottom interpretation strip

**Visual Style:**

- Hand-drawn technical sketch aesthetic
- Professional engineering documentation style
- White background with obvious left-to-right reading order
- Dense annotations but very clear role boundaries
- More like a workflow-control anatomy page than a crowd-of-agents poster
- Composition ratio:
  - 55% detailed schematic illustration
  - 20% labels and technical callouts
  - 25% whitespace

**Color Palette:**

- Deep navy (#2C3E50) for outlines, titles, and workflow frames
- Bright blue (#3498DB) for role-to-role arrows and structural flow
- Teal (#16A085) for clean role separation, successful review, and stable orchestration
- Orange (#E67E22) for overload, interference, and hard-to-debug warnings
- Light gray for helper lines, faded secondary notes, and comparison scaffolding
- Pure white background

**Typography:**

- All annotations in simplified Chinese
- Keep only these technical terms in English where appropriate: planning, retrieval, execution, review, Workflow Split, planner, executor, reviewer
- Title should be bold and large
- Secondary labels should be compact but readable on mobile

**Overall Feel:**

- Clear and highly readable at a glance
- Technical but not abstract
- Makes the difference between "everything in one agent" and "workflow with separated roles" visually unforgettable
- Immediately communicates that workflow design is the real engineering object, and multi-agent is only one implementation form
- Suitable for Xiaohongshu-style knowledge carousel fifth slide

Style: Detailed technical hand-drawn infographic, overloaded single-agent board vs structured workflow decomposition board, role-separated orchestration, pure white background
