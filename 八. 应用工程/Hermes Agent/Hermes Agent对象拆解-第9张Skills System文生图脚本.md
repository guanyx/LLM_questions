# Hermes Agent对象拆解：第 9 张Skills System文生图脚本

Create a highly detailed technical infographic for social media in hand-drawn sketch style on pure white background. The theme is "为什么技能是 Hermes 最像“学习型 Agent”的地方？".

**Design Concept:**
Use a concrete skill-generation-and-reuse journey metaphor instead of a broad method-formation concept diagram: the image should follow one successful task, show how its working method is distilled into a skill file, put into the skill library, and then loaded into a later task. The visual focus should be on one clear before-and-after workflow, not on an abstract “learning” feeling.

**Layout Structure:**

**Top Section (Title Area - 20%):**

- Main title in bold, large font: "为什么技能是 Hermes 最像“学习型 Agent”的地方？"
- Subtitle below: "记忆让 Hermes 记住事实，技能让 Hermes 记住做事方法"
- Small supporting sentence: "Skills 不是普通提示词碎片，而是可发现、可加载、可治理、可生长的程序化记忆"
- Typography should feel like a technical system poster, not a knowledge-management app cover

**Central Section (Hero Diagram - 55%):**
Create a left-to-right concrete skill journey diagram with one “第一次做” and one “下一次复用”:

- Build one thick main path across the center:
  - 一次复杂任务完成
  - 提炼成功步骤
  - 写成 skill 文件
  - 放入 Skill 库
  - 下次任务按需加载
  - 进入当前任务上下文

- Stage 1 - 第一次做成
  - on the far left place one completed task card
  - inside show a tiny example:
    - "修复部署脚本 + 排查依赖问题"
  - add 3 tiny clues around it:
    - 过程复杂
    - 步骤可复用
    - 结果可靠
  - annotation: "不是每个任务都会变技能，但复杂且成功的方法值得沉淀"

- Stage 2 - 提炼方法
  - place one transformation box labeled "经验提炼"
  - inside show 4 compact steps:
    - 提炼步骤
    - 抽象方法
    - 去掉任务偶然性
    - 保留可复用骨架
  - make this look like a加工台，而不是抽象实验室

- Stage 3 - skill 文件
  - place one enlarged example skill card after the transformation box
  - label it: "new skill.md"
  - show 4 tiny fields on the card:
    - 名称
    - 用途
    - 平台匹配
    - 加载条件
  - keep "frontmatter" as a tiny side label instead of main visual center
  - annotation: "技能不是一句提示词，而是带规则的能力单元"

- Stage 4 - Skill 库
  - place one storage shelf / library box after the skill card
  - show multiple stored skill cards inside
  - add two side actions:
    - 修补 / 更新
    - Hub 安装 / 外部加载
  - make it clear this is a living library, not a dead archive

- Stage 5 - 下一次任务
  - on the right place one new incoming task card
  - tiny example:
    - "再次处理类似部署问题"
  - connect it first to a small filter/checkpoint box labeled:
    - 平台匹配
    - 条件判断
    - 安全检查

- Stage 6 - 按需加载进当前任务
  - after the filter box, draw one arrow from Skill 库 into a box labeled "当前任务上下文"
  - add a note:
    - "不是每次全量灌入，而是按需展开"
  - this should visually complete the full before-and-after journey

- Add one green design-strength callout near the middle:
  - label: "设计亮点"
  - annotation: "技能把一次性成功经验变成长期可复用的方法库，Hermes 因此更像会学习的 Agent"

- Add one orange design-tradeoff callout near the library:
  - label: "设计代价"
  - annotation: "技能会生长，就必须治理；否则会膨胀、重复、过时，反过来拖累稳定性"

- Add one bold judgment note below the main path:
  - "技能不是模板库，而是一次做成的方法，下一次按需复用"

**Bottom Section (Interpretation Strip - 25%):**

- Create a horizontal 3-step interpretation strip:
  1. 不是提示词碎片
  2. 是程序化记忆
  3. 会生长，也要治理
- Under each step add one compact explanation:
  - 不是提示词碎片："不是临时 copy 一段文本"
  - 是程序化记忆："有目录、元数据、匹配条件、加载规则"
  - 会生长，也要治理："能沉淀方法，也会带来膨胀风险"
- Add one conclusion sentence centered at bottom:
  "Skills System 让 Hermes 不只是记住你说过什么，而是逐渐记住事情该怎么做"
- Add footer label:
  "Hermes Agent 对象拆解 / 09"

**What Must Be Clear in the Diagram:**

- Skills are not the same as ordinary prompt snippets
- One successful task should visibly turn into one reusable skill
- The skill library is structured and governed
- Skills are loaded on demand, not blindly injected every time
- The image must visually communicate this judgment:
  "技能不是模板库，而是一次做成的方法，下一次按需复用"

**Suitable Diagram Types to Combine:**

- Concrete before-and-after journey diagram
- Experience-to-skill transformation box
- Enlarged skill-file card
- Skill-library shelf
- On-demand loading checkpoint
- Governance/filter strip
- Bottom interpretation timeline

**Visual Style:**

- Hand-drawn technical sketch aesthetic
- Pure white background
- Strong process flow from left to right
- Dense callouts, but visually clean and easy to follow
- Should feel like a concrete “how a skill is born and reused” diagram, not a content library poster
- Composition ratio:
  - 50% detailed schematic illustration
  - 25% text labels and technical callouts
  - 25% whitespace

**Color Palette:**

- Deep navy (#2C3E50) for outlines, titles, frames, and storage structure
- Bright blue (#3498DB) for method-flow arrows and loading paths
- Green (#16A085) for successful task completion and reusable skill growth
- Orange (#E67E22) for governance/tradeoff callouts
- Light gray for helper lines, metadata strips, and secondary structure
- Pure white background

**Typography:**

- All annotations in simplified Chinese
- Keep technical terms in English where appropriate: Skills System, Skill 库, frontmatter, Hub, skill, current task context
- Title should be bold and visually strong
- Metadata labels should be small but readable on mobile

**Overall Feel:**

- Concrete and developmental
- Makes Hermes feel like a system that accumulates know-how through reuse
- Visually emphasizes one clear method-reuse journey, not vague intelligence claims
- Easy to scan, but rich enough for close reading
- Suitable for Xiaohongshu-style technical carousel

Style: Detailed technical hand-drawn infographic, concrete skill birth-and-reuse journey, structured skill-file to library to current-task workflow, pure white background
