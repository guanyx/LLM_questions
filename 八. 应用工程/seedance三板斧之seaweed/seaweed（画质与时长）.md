# Seaweed：把视频做长、做清晰的工程解法（DiT + Video Tokenizer）

> **核心观点**：Seaweed 要解决的不是“让视频更会动”，而是两个更硬的工程目标：**更高保真**与**更长时序**。它的关键抓手通常来自两条线：用 **DiT（Diffusion Transformer）**提高可扩展性，用 **Video Tokenizer（视频压缩表征）**把长视频的计算量压到可训练的规模。

注：关于 Seaweed 的公开细节有限。下文会刻意区分三类表述：

- **公开事实**：能从公开材料直接确认
- **通用做法**：视频生成领域常见的工程路径
- **合理推断**：在信息不足时给出的实现推断，会明确标注为推断

## 1. 为什么光有 PixelDance 还不够？

PixelDance 把“动作与约束”这件事做得更可控，但如果目标是“像成片一样清晰、而且能跑很久”，会撞上更硬的算力与表征瓶颈。

即使不谈具体模型名，只看视频生成的计算结构，也有两类典型天花板：

1. **分辨率瓶颈（通用）**：分辨率越高，特征图越大，训练/推理的显存与算力都会快速增长。
2. **时长瓶颈（通用）**：时长越长，token 数越多，任何需要跨时间建模的模块（尤其是注意力）成本都会急剧上涨。

Seaweed 这类模块的定位，就是把“画质”和“时长”这两个指标往上推，同时把成本压到可落地。

## 2. 核心架构：为什么会转向 DiT（Diffusion Transformer）

一个常见趋势是把扩散的主干网络从 U-Net 迁移到 Transformer 形态（DiT）。这里的关键不是“Transformer 更高级”，而是它在大规模训练下往往更容易出现可扩展性。

### 2.1 为什么要换架构？

- **U-Net（通用观察）**：擅长局部纹理与边缘，但当你要捕捉“长距离依赖”（跨很多帧的一致性、长程叙事、全局构图稳定），往往需要更复杂的结构与训练技巧。
- **DiT（通用观察）**：把输入表征切成 token 序列后，用注意力机制建模全局关系，训练规模上去后更容易表现出可扩展性。

更工程化地说，DiT 转向通常意味着三件事：

1. **Token 化**：把视频从像素网格变成 token 序列，成为 Transformer 的输入单位。
2. **全局交互**：注意力允许 token 之间做全局信息交换，利于建模长程一致性。
3. **规模收益**：在足够数据与算力下，性能提升往往更可预期，但这不是“线性提升”，依然受 token 数、优化稳定性与数据质量制约。

### 2.2 3D 时空注意力 (3D Spacetime Attention)

长视频不崩坏的关键不在于“有注意力”本身，而在于：**如何在可承受成本下，让模型建立跨帧一致性**。

一个常见的实现路线（通用做法）是把时空建模拆开或近似：

- **空间建模（Spatial）**：保证单帧的局部与全局结构合理（清晰度、纹理、边缘）。
- **时间建模（Temporal）**：让跨帧变化更连续，降低身份漂移与纹理抖动的概率。

需要强调：时序注意力不“保证”身份一致，它只是给模型提供了跨帧对齐与信息复用的通道。是否稳定，强依赖训练数据、条件约束强度、以及 tokenizer 的时序一致性。

## 3. 视觉压缩：Video Tokenizer 把长视频变成可训练问题

要生成高分辨率长视频，最大的现实问题是：token 数太多，注意力与显存都会爆。
因此几乎所有可落地的方案都会引入 **Video Tokenizer**（通用做法）：把视频压缩成更小的表征，扩散/Transformer 在压缩表征上生成，再解码回像素空间。

### 3.1 先把问题算清：为什么不压缩根本跑不动

把一个视频当作 `T×H×W×C` 的张量。只要你让模型在这个空间里做全局建模，成本会被三个维度同时放大：

- **分辨率上去**：`H×W` 线性变大，注意力的 key/value 数量也随之变大。
- **时长上去**：`T` 线性变大，你要么让注意力跨更多帧，要么就必须做近似。
- **长视频的“最贵部分”不是一次前向，而是要做很多步扩散/去噪**：即使每一步略快，总步数乘起来仍然很重。

Tokenizer 的作用不是“让模型更聪明”，而是把生成空间从“像素级大张量”改成“可控大小的压缩表征”，让训练与推理变成一个可预算的问题。

### 3.2 Tokenizer 到底压缩了什么

一种常见实现是 **3D VAE / 时空 VAE**（通用做法）。它通常同时压缩三个维度：

- **空间压缩**：把 `H×W` 下采样到更小的 `H'×W'`。
- **时间压缩**：对 `T` 也做下采样或分块编码，得到 `T'`。
- **通道变换**：把像素通道 `C` 映射到 latent 通道 `D`，让表征更适合生成。

因此你可以把它理解成：

- 输入：像素视频 `x ∈ R^{T×H×W×3}`
- 编码：`z = Enc(x)`, `z ∈ R^{T'×H'×W'×D}`
- 生成：在 `z` 空间里做扩散/去噪
- 解码：`x̂ = Dec(z)` 回到像素

关键点在于：**Seaweed 这类模块很大概率是在生成 `z`，而不是直接生成 `x`**。

### 3.3 Tokenizer 决定“你能到多清晰、多稳定”的上限

Tokenizer 的失真往往比你想象中更“结构化”，而且会被后续模型继承：

- **空间方向**：细节被抹平（高频缺失）、边缘发糊、纹理变成“油画感”。
- **时间方向**：轻微拖影、鬼影、周期性闪烁、跨帧纹理不一致。

这也是为什么说 tokenizer 决定上限：如果 `Dec(Enc(x))` 本身就有明显闪烁或拖影，那么后面的 DiT 再强，也很难稳定地生成“比 tokenizer 自己重建还稳定”的视频。

### 3.4 一眼看懂权衡：压缩比、画质、时长之间怎么换

你可以把 tokenizer 的设计视为在控制一个“码率预算”（通用视角）：

- **压缩更狠**（更小的 `T'、H'、W'`）：token 数更少，长视频更可行，但细节与时序一致性更难保。
- **压缩更轻**（更大的 `T'、H'、W'`）：单帧更清晰、局部更真实，但 token 数上去后注意力/显存会先炸。

工程上最实用的经验是先做两个测试：

1. **Tokenizer 重建测试**：只看 `Dec(Enc(x))` 的清晰度与稳定性。
2. **规模敏感测试**：同一个 DiT 结构，在不同 `T'、H'、W'` 下能跑到什么 batch、什么时长。

只要这两项跑通，你才有资格讨论后面“训练配方能不能再提升”。

## 4. 训练策略：画质与稳定性如何被“优化目标”表达出来

“美学”不是一个训练目标，它必须被落到数据与损失上。

在公开信息不足的情况下，更稳妥的表达是列出常见做法（通用做法）：

### 4.1 数据侧：先把“会污染目标”的分布剔除

画质与稳定性的下限，首先由数据决定。对长视频尤其如此，因为模型会把“视频里最常见的模式”当作更省损失的解。

- **清晰度与压缩过滤**：低码率块效应、强降噪、过锐化会把伪影变成“纹理规律”。
- **镜头属性过滤**：快切转场、闪白闪黑、强运动模糊会鼓励模型学到“抖动式变化”。
- **水印与贴纸过滤**：一旦进入训练集，模型很容易把它当成可复用模式。
- **长时序一致性筛选（更关键）**：同一主体跨帧是否稳定、是否存在突然跳变、是否大量遮挡。

### 4.2 目标侧：让“清晰、稳定”在损失里有梯度

扩散模型的主损失通常是噪声预测或速度预测的回归损失，但这不等于它自然会得到“清晰且稳定”。常见补强手段（通用做法）包括：

- **感知相关损失**：让细节更接近人眼感受，而不是只追 MSE。
- **时序一致性约束**：对相邻帧或短窗内的特征一致性施加约束，减少闪烁。
- **结构一致性约束**：对边缘、深度、分割等辅助表征做一致性约束，减少主体漂移。

它们的共同点是：不直接定义“美学”，而是把“清晰、稳定”拆成可微的、可优化的信号。更具体地说，工程上通常按下面的方式落地。

#### 4.2.1 感知相关损失：用“特征距离”替代“像素距离”

做法是引入一个冻结的视觉特征提取器 `φ(·)`（通用做法：VGG/CLIP/LPIPS 特征网络等），用特征空间距离衡量生成结果与真值的差异。

- **直觉**：像素 MSE 会奖励“平均化”的解，而特征距离更倾向保留边缘、纹理与局部结构。
- **形式**：`L_perc = Σ_l || φ_l(x̂) - φ_l(x) ||`，对多个层级的特征做加权。
- **放到扩散训练里怎么接**：模型每一步预测的是噪声/速度，但你可以把它换算成对“干净样本”的估计 `x̂0`（或 latent `ẑ0`），再对 `x̂0`（解码后）计算感知损失。

关键的工程权衡在于成本：感知损失往往需要把 `x̂0` 解码回像素并过一次特征网络，所以常见的降成本方式是只在少量时间步计算、只对低分辨率版本计算，或把它放到最后的精修/超分阶段。

#### 4.2.2 时序一致性约束：把“少闪烁”变成可优化目标

减少闪烁的难点在于：相邻帧本来就应该不同，你不能简单要求 `x̂_t` 和 `x̂_{t+1}` 相等。常见做法是“对齐后再比较”：

- **光流对齐（常见）**：先算出“这一帧里的每个位置，在下一帧大概移动到了哪里”（光流 `F_{t→t+1}`）。然后按这个移动关系，把 `x̂_t` 里的内容挪到 `t+1` 的位置上（warp），让两帧先对齐，再去比较它们的差异：`|| warp(x̂_t, F) - x̂_{t+1} ||`。这样主要惩罚的是对齐后仍然不一致的部分，更接近我们说的闪烁/漂移。
- **特征对齐（更稳）**：不是拿两帧的“每个像素值”去硬对比，而是先用一个网络 `ψ(·)` 把图像变成更抽象的特征（更像轮廓、结构、语义这种信息），再去比较这些特征是否一致：`|| ψ(warp(x̂_t)) - ψ(x̂_{t+1}) ||`。这样对轻微的颜色抖动、纹理噪声不那么敏感，更容易把真正的时序不稳定（闪烁/漂移）抓出来。
- **短窗一致性（更贴合长视频）**：不只约束 `t,t+1`，而是在一个短窗口内约束一致性，减少低频闪烁。

这里真正起作用的是“对齐”而不是“相等”。对齐把“该动的运动”解释掉，留下的残差更像是闪烁与漂移。

#### 4.2.3 结构一致性约束：让主体几何与边界更稳

结构一致性的思路是：选一类你真正关心的结构表征 `s(·)`，让它在生成结果与真值之间更一致。

- **边缘/线稿一致性**：用 Sobel/Canny 或可学习的边缘提取器得到边缘图，对边缘图做 L1/BCE。
- **深度一致性**：用冻结的深度网络从 `x̂` 和 `x` 预测深度图，约束深度差异，减少“主体忽大忽小”。
- **分割/人体解析一致性**：用冻结的分割/人体解析网络得到 mask 或部件标签，约束语义边界，减少主体漂移。

这类约束本质是在告诉模型：你可以在纹理上自由发挥，但在“几何/边界”上不要随意漂。

把这些信号加起来，训练目标通常会变成一个加权和：`L = L_diff + λ1 L_perc + λ2 L_temp + λ3 L_struct`。系数的意义很简单：你愿意为“更清晰/更稳定”付出多少训练难度与算力成本。

### 4.3 偏好对齐：当损失不够表达“好看”时怎么办

仅靠训练损失往往仍然不能把“成片级观感”拉到足够高。这时更常见的是引入偏好信号（通用做法）：

- **打分器筛选/重采样**：生成多条候选，用审美/清晰度/稳定性打分器挑更好的。
- **偏好数据再训练**：用人类偏好或规则偏好构造对比样本，做偏好优化或蒸馏。

注意这里的关键不是“加一个打分器就好了”，而是：你必须能持续产出可靠的偏好数据，否则会把模型推向不可控的局部最优。

### 4.4 稳定性工程：长视频训练为什么更容易崩

同一套损失，在长视频上更难收敛的原因通常不是“模型不够大”，而是优化过程更不稳定（通用观察）：

- **更长的依赖链**让梯度更噪、更难对齐。
- **token 数更多**让 batch 更小，统计不稳定。
- **训练分布更复杂**使得模型更倾向学到“捷径”（比如抖动、转场）。

因此训练配方往往会更强调稳定性：更稳的 EMA、更严格的梯度裁剪、更保守的学习率与噪声调度，以及更明确的 curriculum（先短后长、先低分辨率后高分辨率等）。

如果未来能拿到 Seaweed 的公开技术细节，这一节应当把“它具体用了什么 tokenizer/什么注意力近似/什么训练配方”单独列为公开事实。

## 5. 端到端数据流：Seaweed 在系统里到底做哪一段

如果用纯文字复述一次可落地的系统流程（通用形态），Seaweed 更像“高保真长时序渲染器/精修器”：

### 5.1 推理时（用户调用一次）发生什么

1. **整理条件输入**：把文本提示词、首尾帧/关键帧、姿态/草图/分割等控制信号统一到一个条件包里。
2. **把像素条件变成可用表征**：首尾帧/关键帧这类像素条件，通常也会被编码到某种 latent 或特征空间，方便与 DiT 融合。
3. **初始化要生成的 latent 序列**：在 `z ∈ R^{T'×H'×W'×D}` 空间里初始化噪声，准备做扩散去噪。
4. **逐步去噪生成**：DiT 在条件约束下迭代去噪，得到一段长时序 latent。
5. **解码回像素视频**：`x̂ = Dec(z)`，必要时再做超分、去噪、细节增强等后处理。

### 5.2 训练时（模型能力的来源）发生什么

1. **数据预处理**：解码视频、统一帧率/分辨率、切分镜头、剔除污染样本，生成可训练片段。
2. **Tokenizer 训练或冻结**：先把 tokenizer 训练到足够好的重建质量，然后固定或半固定它，保证生成目标稳定。
3. **构造训练对**：把像素视频编码成 `z`，同时准备对应的条件（文本、关键帧、控制信号）。
4. **扩散训练**：对 `z` 加噪，在条件下让 DiT 预测噪声/速度并回归；必要时叠加清晰度与一致性相关的辅助目标。

### 5.3 它和 PixelDance 怎么接起来

在一个更像产品的系统里（通用形态）：

- **PixelDance 提供“强约束”**：比如首尾帧、关键帧、动作/姿态轨迹，让“怎么动”更可控。
- **Seaweed 提供“高保真长时序生成”**：把同样的约束投到压缩表征空间里，以更可扩展的方式生成更长、更清晰、更稳定的视频。

这也是模块化最直接的工程收益：动作控制与画质/时长提升，不必强耦合在同一条训练链路里。

这条链路解释了 Seaweed 的核心价值：它不是“更会讲动作”，而是把长视频的计算问题变成“在压缩空间里可训练、可扩展的生成”。

## 6. 总结：模块化解耦，职责清晰

回到 Seedance 2.0 的全景图，更合理的分工表述是：

- **PixelDance** 更偏“约束与动作”：把生成问题改写成带强边界条件的可控问题。
- **Seaweed** 更偏“画质与时长”：用 tokenization 降维，用 DiT/注意力建模提升可扩展性，尽量把视频做清晰、做稳定、做长。

模块化解耦的工程价值在于：你可以分别优化“可控性”和“高保真长时序”，再在接口处组合，而不是用一个黑盒同时顶住所有目标。
