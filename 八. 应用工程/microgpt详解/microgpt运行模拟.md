# MicroGPT 运行模拟：追踪数据的旅程

与其阅读枯燥的代码定义，不如让我们把代码跑起来。我们将追踪**具体的数据**是如何在 MicroGPT 中流动、变形，最后变成智能的。

---

## 第一幕：Autograd 实验室（Value 的前世今生）

假设我们有一个极其简单的任务：让 `a + b` 的结果变小。
初始状态：`a = 2.0`, `b = -3.0`

### 1. 前向传播（Forward）：记账
代码运行：`c = a + b`
此时内存中发生的事情：
*   **创建 c**：`c.data` 变为 `-1.0` (2 + -3)。
*   **记录家谱**：`c._children` 记住了 `(a, b)`。`c` 知道自己是 `a` 和 `b` 的孩子。
*   **记录局部导数**：`c` 记住了“如果 `a` 变大 1，我也变大 1”（加法的导数是 1）。

代码运行：`L = c * c` (Loss，我们希望它接近0)
*   **创建 L**：`L.data` 变为 `1.0` (-1 * -1)。
*   **记录家谱**：`L` 记住了 `(c)`。

### 2. 反向传播（Backward）：追责
现在我们调用 `L.backward()`。目标是：**为了让 L 变小，a 应该怎么变？**

1.  **L (1.0)**：
    *   `L.grad` = 1.0 (基准)。
2.  **L -> c (-1.0)**：
    *   L 问：c 变大一点，我会变大多少？
    *   数学说：$L = c^2$，导数是 $2c = -2$。
    *   所以 `c.grad` = $1.0 \times -2 = -2.0$。
    *   含义：**c 变大，L 会变小（因为梯度是负的）。**
3.  **c -> a (2.0)**：
    *   c 问：a 变大一点，我会变大多少？
    *   数学说：$c = a+b$，导数是 1。
    *   所以 `a.grad` = `c.grad` $\times 1$ = $-2.0$。
    *   含义：**a 变大，L 也会变小。**

### 3. 更新（Update）
优化器说：既然 `a.grad` 是负的（-2.0），说明 a 变大能减小 Loss。那我们就把 a **增大**一点点！
`a` 从 `2.0` 变成了 `2.01`。

## 第二幕：Attention 会议室（单词 "Hello" 的内部沟通）

现在我们模拟 GPT 处理单词序列 `"He"` 的过程。
假设嵌入维度极简为 2（真实是 16）。

### 场景：
模型正在读第 2 个字 **"e"**。
它需要决定：我是只关注我自己，还是回头看看前面的 **"H"**？

### 1. 准备数据（Embedding）
*   **"H" (位置0)** 的向量：`x0 = [1.0, 0.5]`
*   **"e" (位置1)** 的向量：`x1 = [0.0, 2.0]`

### 2. 生成 Q, K, V
模型通过矩阵乘法（`linear`）把向量变成了 Query, Key, Value。假设结果如下：
*   **"e" 的查询 (Q1)**：`[1.0, 0.0]` （意思是：我在找第1个维度是正数的字）
*   **"H" 的名片 (K0)**：`[1.0, 0.0]` （正好！我的第1维度是 1.0）
*   **"e" 的名片 (K1)**：`[0.0, 1.0]` （我的第1维度是 0）
*   **"H" 的内容 (V0)**：`[5.0, 5.0]`
*   **"e" 的内容 (V1)**：`[1.0, 1.0]`

### 3. 计算匹配度（Dot Product）
"e" 拿着自己的 Q1，去和历史记录里的 K 碰一碰：
*   **vs "H"**：$Q1 \cdot K0 = 1.0 \times 1.0 + 0 = 1.0$ (**高分！**)
*   **vs "e"**：$Q1 \cdot K1 = 1.0 \times 0.0 + 0 = 0.0$ (低分)

### 4. 归一化（Softmax）
把分数变成概率：
*   "H": `exp(1.0) ≈ 2.7` -> **73%**
*   "e": `exp(0.0) = 1.0` -> **27%**

### 5. 融合信息（Weighted Sum）
"e" 最终输出的向量，是 V0 和 V1 的加权混合：
`Output = 0.73 * [5, 5] + 0.27 * [1, 1] ≈ [3.9, 3.9]`

**结论**：
虽然当前字是 "e"，但经过 Attention 后，它融合了大量 "H" 的信息（因为 Q 和 K 匹配）。这就是 GPT **“理解上下文”** 的微观过程。

## 第三幕：训练场（猜词游戏）

### 1. 预测（Logits）
经过上面的处理，模型对 "He" 后面是什么字给出了打分（Logits）：
*   "l": 5.0 分
*   "o": 1.0 分
*   "x": -2.0 分

### 2. 概率（Softmax）
*   "l": 98%
*   "o": 1.5%
*   ...

### 3. 判卷（Loss）
真实答案是 "l" (Hel**l**o)。
*   模型给了 "l" 98% 的概率。
*   Loss = `-log(0.98) ≈ 0.02`。
*   **Loss 很小**，老师摸摸模型的头，不用怎么改。

**如果真实答案是 "o" (He**r**o) 呢？**
*   模型只给了 "o" 1.5% 的概率。
*   Loss = `-log(0.015) ≈ 4.2`。
*   **Loss 巨大！** 反向传播启动！
    *   所有导致 "l" 分数高的参数，都会被**狠狠惩罚**（梯度大，被大幅修改）。
    *   所有导致 "o" 分数低的参数，都会被**奖励**（往反方向修改）。

下次再遇到 "He"，模型就会更倾向于猜 "o" 了。这就是**训练**。
