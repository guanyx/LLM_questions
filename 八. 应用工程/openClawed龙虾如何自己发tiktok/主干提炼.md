# 用 AI Agent 自动化做 TikTok 图文轮播：一套可复用的打法（主干提炼）

这套方法的核心不是“让 AI 生成几张图”，而是把一个可迭代的内容工厂拆成可自动化的环节：选题与钩子、素材生成、排版合成、发布编排、复盘学习。人只保留暂时无法 API 化、但对分发影响最大的“最后一公里”（比如选热门音乐）。

## 一句话结果

- 内容形态选 TikTok Photo Carousel（图文轮播/幻灯片），按固定规格批量生产
- 用 Agent 跑流程：研究 -> 生成 6 张同一场景的“改造前后” -> 叠字 -> 写 caption -> 通过 API 先发草稿
- 人工只做：选 trending sound、粘贴 caption、点发布（约 60 秒/条）
- 成功来自两个杠杆：可复用的“钩子公式” + 持续迭代的“技能/记忆文件”

## 为什么选“图文轮播”

- 轮播是更易工业化的内容形态：不需要剪辑，不依赖真人出镜
- 它天然适配“第一张图 + 钩子字”的停留机制，能用最小成本测试大量 hook
- 对工具链友好：生成图片、叠字、排版、上传都更容易自动化

## 系统拆解：你需要什么

### 1) 一台长期在线的机器

- 低配也能跑（2GB 内存可用，4GB 更稳；1-2 vCPU；20GB SSD）
- 可以是闲置电脑、树莓派、VPS（上 VPS 要有基本安全意识）

### 2) 一个能“用工具”的 Agent 框架

关键能力不是聊天，而是：
- 有身份与长期目标
- 能读写文件（保存技能与记忆）
- 能调用外部工具/API（图像生成、排版、发布）

### 3) 三类能力模块

- 图像生成：用外部图像模型生成高拟真图片
- 叠字/合成：把 hook 文案叠到第 1 张图，保证手机端可读
- 发布：通过社媒工具或自建脚本把内容上传到 TikTok（最好能创建草稿）

### 4) 两类“文本资产”（最关键）

- Skill files：把流程写成可执行的规则与模板（像培训新人一样写）
- Memory files：把每次发布结果、踩坑与结论记录下来，供后续决策引用

## 内容规格：固定约束让产能稳定

建议把每条内容固定成一份“简报”，让 Agent 永远按同一结构执行：

- 幻灯片数量：6 张
- 第 1 张：必须叠 hook（短、清晰、可读）
- 图片尺寸：竖版 1024x1536（避免黑边）
- caption：讲一个小故事，让产品自然出现（而不是硬广）
- hashtag：最多 5 个（控制数量，避免噪声）

## 生成图片的关键：锁定“建筑结构”，只换“风格”

如果你的内容是“同一个房间/同一个对象的多种风格改造”，最大的坑是：每张图变成不同场景，观感立刻穿帮。

解决思路：把“不可变部分”写到极致细，复制到每一张图的 prompt 里；每张图只改“风格变量”。

### Prompt 模板（可复用）

把下面分成两段：A 固定不变，B 每张变一次。

**A. 场景锁定（每张完全一致）**
- 拍摄设备与质感：iPhone photo / natural phone camera quality / realistic lighting
- 构图：拍摄位置、镜头方向、视角高度、焦段感
- 空间结构：长宽、高度、门窗数量与位置、墙体关系
- 家具与关键物件：尺寸、摆放位置、数量
- 材质：地板、墙面、天花、灯光类型
- 生活痕迹：杯子、遥控器、小物件（避免“样板间”空镜感）

**B. 风格变量（每张只改这一段）**
- 目标风格：现代/北欧/乡村/极简/复古…
- 颜色与材料：墙色、橱柜材质、五金颜色、织物纹理
- 装饰与软装：摆件、灯具、植物、挂画、床品

经验要点：
- 早期“描述模糊”会导致每张图结构漂移（窗户移动、柜体换边）
- “before”不要破败，要“还行但疲惫”，更贴近日常，代入感更强

## 发布链路：先草稿，后人工补音乐

音乐对分发影响巨大，但很多发布 API 不支持选音乐或无法保证效果一致。

推荐流程：

1. Agent：生成 6 张图（同结构不同风格）
2. Agent：给第 1 张叠 hook（确保手机端可读）
3. Agent：写 caption（故事化 + 自然带出产品）
4. Agent：通过 API 上传为 TikTok 草稿（隐私设为仅自己可见）
5. Agent：把 caption 单独发给你（方便粘贴）
6. 你：打开 TikTok 草稿，选 trending sound，粘贴 caption，发布

这样把 95% 的体力活交给 Agent，同时保留对音乐与最终质感的掌控。

## 最重要的增长点：钩子（Hook）公式

最容易失败的 hook：讲自己、讲功能、讲对比（“我们这个很牛”）。

更稳定的 hook：让观众先在脑中生成一个“人类时刻”。

### 可复用公式

> [另一个人] + [冲突/质疑] → 我把 AI 结果给 TA 看 → TA 改变想法/被震撼

把“产品”放在故事里当工具，而不是主角。

示例结构（替换括号即可）：
- 我的[房东/对象/妈妈/室友]说[不能改/不值得/太贵]，直到我给 TA 看了 AI 做的这个
- 我不敢[装修/动手/花钱]，先用 AI 把[同一个空间]试了 6 种风格

## 让系统越来越强：把复盘写进文件

很多人的 AI 自动化卡在“每次都从零开始”。可复用的关键在于把失败变成规则，把成功变成模板，让 Agent 的产出随时间复利。

### Skill file 应该包含什么

- 内容规格：尺寸、张数、格式、哈希标签上限
- 叠字规范：字号比例、位置（避开状态栏）、行宽限制
- Prompt 模板：场景锁定段 + 风格变量段
- Hook 库：高表现 hook 的模板与变体
- Caption 模板：故事结构与产品植入方式
- 失败清单：每次踩坑都写进去，保证“不再犯”

### Memory file 记录什么

- 每条内容：hook、风格组合、发布时间、播放/互动、转化（如果能拿到）
- 结论：哪些 hook 形式更稳、哪些视觉更像“手机实拍”、哪些叠字不可读
- 下一轮实验计划：要验证的变量、预期结果、成功标准

## 常见失败与修复清单

- 黑边/比例不对：改为 1024x1536 竖版
- 场景不一致：把“结构描述”写到足够细，并在每张完全复用
- 字太小/被遮挡：增大字号比例，下移位置，控制行宽避免挤压变形
- hook 没人点：把“我/我的产品”从主语移走，改成“另一个人 + 冲突”
- 本地生成质量不够：优先保证拟真度与一致性，成本往往低于时间损耗

## 从 0 到 1 的搭建步骤（最短路径）

1. 准备一台常在线机器，部署 Agent 框架与基础工具权限
2. 接好两类外部能力：图像生成、草稿发布（API）
3. 写第一版 Skill file：把规格、模板、叠字规则先定死
4. 一次性 brainstorm 10-15 个 hook，选 3-5 个先跑
5. 运行：夜间批量生成 -> 白天你只负责选音乐发布
6. 每天复盘：把失败写成规则，把成功写成公式，更新 Skill/Memory

这套方法的“真正解锁点”不是某个模型，而是你围绕 Agent 构建的可迭代系统：流程固化、规格稳定、钩子可复用、复盘能沉淀。只要这些成立，产出会越来越好，单位内容成本会越来越低。
