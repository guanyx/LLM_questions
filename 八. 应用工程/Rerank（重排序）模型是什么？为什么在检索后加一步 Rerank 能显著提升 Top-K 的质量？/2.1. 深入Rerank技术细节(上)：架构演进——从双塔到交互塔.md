# 深入 Rerank 技术细节(上)：架构演进——从双塔到交互塔

上一篇文章我们聊了 Rerank 的“图书管理员”角色。今天，我们关上图书馆的大门，走进后台的技术实验室，拆解一下 Rerank 到底是怎么工作的。

我们将首先探讨 Rerank 技术的基石——架构演进。从最基础的**Bi-Encoder**（双塔），到 Rerank 的本尊**Cross-Encoder**（交互塔），再到为了平衡速度与精度而诞生的折中派**Late Interaction**（如 ColBERT）。

---

## 01. 架构之争：为什么 Cross-Encoder 比 Bi-Encoder 准？

我们在做检索时，最常听到的词是 Embedding（向量化）。但同样是用 BERT/Transformer 模型，怎么用法决定了它是“快而不准”还是“准而不快”。

### Bi-Encoder（双塔架构）：初筛的王者
*   **别名**：Dense Retriever, Embedding Model
*   **工作流**：
    1.  把 Query 输入模型 A，得到向量 $V_q$。
    2.  把 Doc 输入模型 B（通常 A=B），得到向量 $V_d$。
    3.  计算 $V_q$ 和 $V_d$ 的余弦相似度（Cosine Similarity）。
*   **核心特征**：**“不见面”**。
    Query 和 Doc 在变成向量之前，完全不知道对方的存在。它们唯一的交互是在最后一步算分时做一个简单的点积。
*   **技术瓶颈**：**信息瓶颈（Information Bottleneck）**。
    无论 Query 还是 Doc 有多复杂，最后都被强行压缩成了一个固定维度（比如 768 维）的向量。在这个压缩过程中，词与词之间的精细交互信息（Interaction）全部丢失了。

### Cross-Encoder（交互架构）：Rerank 的标配
*   **别名**：Full-Interaction Model
*   **工作流**：
    1.  把 Query 和 Doc **拼**成一个长句子：`[CLS] Query [SEP] Doc [SEP]`。
    2.  把这个长句子塞进 BERT/Transformer 模型。
    3.  Transformer 内部的 **Self-Attention（自注意力机制）** 开始工作。
    4.  取 `[CLS]` 位置的输出，接一个全连接层（Linear Layer），直接输出一个 0~1 的相关性分数。
*   **核心魔法**：**Attention Map**。
    在 Transformer 的每一层（比如 12 层），Query 里的每一个 token 都在和 Doc 里的每一个 token “握手”（计算 Attention 权重）。
    *   Query 里的“不”字，可以抑制 Doc 里“含糖”这个词的权重。
    *   Query 里的“Python”，可以放大 Doc 里“pip install”的权重。
    这种交互发生了成千上万次（$L \times N \times M$），没有任何信息被压缩，所有细节都被模型“看”在眼里。

**结论**：Cross-Encoder 的精度优势，来自于它允许 Query 和 Doc 在**每一层神经网络中**都进行深度交互，而不是像 Bi-Encoder 那样只在最后时刻碰一下头。

## 02. 鱼与熊掌：ColBERT 与 Late Interaction

Cross-Encoder 虽好，但太慢了。每来一个 Query，都要和 50 个 Doc 拼在一起跑 50 次大模型推理。能不能既要有 Bi-Encoder 的速度（预计算 Doc），又要有 Cross-Encoder 的精度（细粒度交互）？

**ColBERT (Contextualized Late Interaction over BERT)** 应运而生。它是近年 RAG 界的当红炸子鸡。

### 它的思路是：别把 Doc 压成一个向量，压成一袋子向量！

1.  **Bi-Encoder 做法**：Doc -> `[v_doc]` (1个向量)。
2.  **ColBERT 做法**：Doc -> `[v_word1, v_word2, ..., v_wordN]` (N个向量)。
    *   它保留了文档中**每个 token** 的向量表示（经过上下文编码后的）。
    *   虽然这会让索引变大几十倍（空间换时间），但它保留了细节。

### MaxSim 操作
当 Query 来了（假设 Query 也有 M 个 token）：
1.  对于 Query 中的**每一个** token向量 $q_i$。
2.  去和 Doc 中的**所有** token向量 $d_j$ 算相似度，找出**最大**的那个分数（MaxSim）。
3.  把所有 $q_i$ 的最大分数加起来，就是最终得分。

**为什么这很牛？**
这就好比：不用把整本书压成一句话（Bi-Encoder），也不用通读全书（Cross-Encoder）。而是 Query 里的每个关键词，都去 Doc 里找它**最匹配的那个词**作为代表。
这保留了“词对词”的细粒度交互（Late Interaction），同时 Doc 的向量可以离线算好，查询速度接近 Bi-Encoder。
