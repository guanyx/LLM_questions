# GLM-5 技术白皮书：定义开源 SOTA 新标准

**发布时间**：2026 年 2 月 11 日  
**模型类型**：744B MoE (Mixture-of-Experts)  
**开源协议**：MIT License  
**核心地位**：当前开源界 Coding 与 Agent 领域的 State-of-the-Art (SOTA) 模型，综合能力逼近闭源顶流 Claude Opus 4.5。

本文将从技术架构、基准测试、智能体工程及原生能力四个维度，详细解析为何 GLM-5 被视为当前的开源 SOTA。

## 1. 核心技术架构 (Architecture & Specs)

GLM-5 并非单纯的参数堆砌，而是在模型架构与训练效率上进行了深度优化。

- **参数规模与 MoE 架构**：
  - **总参数量**：744B (7440 亿)
  - **激活参数**：40B (400 亿) / token
  - **技术解析**：MoE 通过路由机制让每个 token 只激活部分专家，在获得超大模型容量的同时，将单次推理的计算量压到“激活参数”这一量级。它通常会显著降低同等总参数稠密模型的推理成本，但 744B 级模型依然更适合多卡/服务化部署。
- **DeepSeek Sparse Attention (DSA)**：
  - **集成创新**：GLM-5 深度集成了 DeepSeek 的稀疏注意力机制 (DSA)。
  - **技术优势**：官方口径是“在保留长上下文能力的同时显著降低部署成本”。从工程直觉看，它主要通过稀疏化注意力计算来提升长上下文的 token efficiency，缓解长上下文下注意力计算与 KV Cache 的成本压力。
- **长上下文能力**：
  - **上下文窗口**：200K tokens（Artificial Analysis 的技术规格项）
- **全栈国产算力验证**：
  - **训练底座**：多家媒体报道其训练使用华为 **Ascend (昇腾)** 芯片，但当前公开仓库与开发者文档未给出可复现级别的训练细节。
  - **SOTA 意义**：这给市场传递了一个强信号：在受限的算力供给环境下，依然能做出在关键榜单上与国际顶级模型同台竞技的模型。
- **训练数据规模（官方披露）**：
  - **预训练数据量**：28.5T tokens（相较前代 23T 增长）

## 2. 基准测试：硬核 SOTA 证据 (Benchmarks)

GLM-5 的“强”体现在其于高难度、复杂推理任务中的统治级表现。

### A. 代码工程 (Coding) —— 击败 Google

在衡量模型解决真实 GitHub Issue 能力的权威榜单 **SWE-bench Verified** 中：

- **GLM-5 得分**：**77.8%**
- **Gemini 3 Pro 得分**：76.2%
- **Claude Opus 4.6 得分**：80.9%
- **结论**：在媒体引用的官方材料口径中，GLM-5 在 SWE-bench Verified 上超过 Gemini 3 Pro，并逼近 Claude Opus 级别成绩，属于“能用跑分解释的硬实力提升”。

### B. 智能体能力 (Agentic) —— 开源第一

在考察模型工具使用与长链路规划的测试中：

- **Vending Bench 2** (模拟经营)：排名 **开源 #1**。展示了在数百步操作中保持策略一致性的能力。
- **MCP-Atlas** (工具调用)：在准确率和多步规划上表现最优，证明其对复杂 API 的理解力远超同类。

### C. 综合智能

- **Artificial Analysis Intelligence Index**：得分 **50**，处于第一梯队。

## 3. 智能体工程 (Agentic Engineering) 技术拆解

GLM-5 最大的技术突破在于从“对话模型 (Chat Model)”向“执行模型 (Action Model)”的范式转变。

### A. 异步强化学习 (Asynchronous RL)

- **技术痛点**：传统 SFT (监督微调) 难以教会模型处理长达数百步的复杂任务，模型容易在中间步骤“迷路”。
- **GLM-5 方案**：引入异步 Agent RL 训练框架。通过大规模的模拟环境交互，让模型在长链路任务（Long-horizon tasks）中学习如何分解目标、管理资源和自我纠错。
- **效果**：实现了在复杂环境下的“目标锁定”，使其能胜任运维、模拟经营等长程任务。

### B. 原生工具调用与协议适配 (Tool Use)

- **能力锚点**：官方材料强调其在 BrowseComp、MCP-Atlas、τ²-Bench 等“工具调用/多步规划”评测中达到开源 SOTA。工程上，这通常意味着：对工具调用格式、参数填充、回填信息利用、多轮状态保持等关键环节进行了专门训练与对齐。
- **部署侧信号**：开源部署脚本中包含 `--tool-call-parser`、`--enable-auto-tool-choice` 等选项，说明其工具调用存在相对固定的结构化协议与解析链路。

### C. Deep Thinking (慢思考模式)

- **机制**：官方 API 支持开启 `thinking`（思考）模式，并在多家推理框架中存在独立的 reasoning 解析器配置。这意味着模型具备“先推理、后作答”的输出形态，但其内部训练细节（例如是否采用某种特定格式的内部独白数据）官方尚未公开。

## 4. 原生能力：办公文档交付（Artifacts）

GLM-5 将“生成能力”从文本扩展到了**企业级交付物 (Artifacts)**，其技术实现路径具有极高的护城河。

- **技术原理（实现细节未完全公开）**：**工具链生成（Tool-assisted Artifacts）**
  多家媒体与官方宣传口径强调 GLM-5 可直接产出可用的 .docx/.pdf/.xlsx 等办公文件。就工程实现而言，这类能力通常不是“直接生成二进制”，而是由模型产出结构化中间表示或步骤计划，再由渲染/模板/转换工具链生成最终文件。
  1.  **需求解析**：将“报告/表格/方案”拆成结构（章节、表格、图表、样式）与数据约束（字段、口径、计算逻辑）。
  2.  **计划与生成**：产出结构化描述（例如 JSON/DSL）或可执行步骤（例如调用渲染器/模板引擎的参数）。
  3.  **渲染与校验**：在工具链侧生成文件，并进行基本校验（格式、公式、缺失字段）。
- **SOTA 体验**：如果上述链路稳定，它会把 LLM 的输出从“可读文本”升级为“可交付工件”，这对办公自动化与企业流程集成的价值显著高于纯文本输出。

## 5. 总结

GLM-5 之所以被称为 **SOTA**，是因为它在以下三个维度重新定义了开源模型的上限：

1.  **上限极高**：在 Coding 和 Agent 领域，它在 SWE-bench Verified 等指标上超过了 Google Gemini 3 Pro，并与行业最顶尖的闭源模型（Claude Opus）处于同一代际。
2.  **架构先进**：744B MoE + DSA 的组合，在算力效率和模型容量之间找到了完美平衡。
3.  **工程落地**：通过异步 RL 和工具链交付，它解决了 LLM 落地企业级场景（长任务、文档交付）的“最后一公里”难题。

**结论**：GLM-5 是目前开源生态中，极少数能同时覆盖**高性能代码生成**、**长链路 Agent 规划**和**企业级文档交付**的 SOTA 级模型。
