1）DSPy 把 LLM 开发从“手写 Prompt 字符串”升级为“声明式编程”：只声明输入/输出/任务描述，Prompt 由编译器自动生成与优化。

2）Signature = 任务接口（Schema）：用 InputField/OutputField 定义字段，Docstring 就是高层 Instruction，替代传统 Prompt 模板。

3）Module = 执行器：Predict 做最基础的输入 → 输出；ChainOfThought 会自动插入 Rationale 字段，把“先推理再回答”变成默认流程。

4）Metric = 成功定义：用规则或裁判模型打分，把“感觉调 Prompt”变成“可量化迭代”，否则 Teleprompter 无法优化。

5）Teleprompter（如 BootstrapFewShot）= Prompt 编译器：Teacher 跑任务 →Metric 过滤高质量 → 收集(Input,Rationale,Output)→ 把 demos 注入 Prompt；也可离线编译后导出（save 或 inspect_history）给 Java/Go/Node 线上直接拼接部署。
