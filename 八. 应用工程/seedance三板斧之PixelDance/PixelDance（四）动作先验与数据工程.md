# PixelDance（四）：动作先验怎么来的？以及为什么 80% 难度在数据

前三章把“可控任务定义”和“多模态条件接口”讲清楚了。剩下两个最关键的问题是：

1.  模型到底怎么“学会动”（动作先验不是玄学）
2.  如果要复刻，为什么工程瓶颈在数据而不是代码

## 1. 动作先验到底是怎么学出来的？

“动作先验”不是一个玄学概念，它可以被拆成一句工程描述：

- **在扩散的去噪损失下，模型为了把噪声从视频 latent 中抹掉，被迫学会了跨时间的相关结构。**

### 1.1 训练目标：为什么优化一次 MSE，就会“学会运动”

扩散训练的目标可以非常粗暴地理解为：对一段视频 latent 加噪后，让网络在条件约束下把噪声预测回来，用均方误差驱动参数更新。

关键点在于：噪声是加在整段视频 latent 上的。要让误差变小，网络必须利用跨帧信息：

- 单帧视角只知道“这一帧像什么”，但不知道“下一帧应该怎么变”
- 跨帧建模能学到稳定的变化模式：人体关节的连续性、物体的惯性、布料的延迟、镜头运动的平滑性

这些“能让损失更低的统计规律”，就是工程意义上的动作先验。

### 1.2 架构手术：让 U-Net 看懂时间

文生图的 U-Net 原生只理解 `(H, W)`。要做视频，最低限度要让网络对 `T` 维度可感知。常见做法是把 2D 网络扩成“2D+时间模块”的形态：

- **空间层（2D Conv / Spatial Attention）**：负责单帧的纹理、边缘、材质
- **时间模块（Temporal Conv / Temporal Attention）**：负责跨帧一致性与变化轨迹

时间模块不是“只看位移不看内容”，它也看内容，但它的参数会更偏向学习：哪些内容在时间上应该保持，哪些内容应该连续变化。

### 1.3 尾帧不是“像素级终点”，而是“概率上的终点”

用户给定的“尾帧”通常是完美的静态图。如果强行让视频每一像素都精准对齐这张图，生成的动作会非常生硬（像机器人归位）。更合理的策略是把尾帧当成“概率终点”：

1.  **尾帧扰动（Perturbation）**：对尾帧条件做轻微扰动，把“精确复制尾帧”降级为“向尾帧分布靠拢”
2.  **尾帧丢弃（Dropout）**：训练时以一定概率丢弃尾帧条件，逼迫模型在没有“终点答案”的情况下仍能生成合理运动轨迹
3.  **两阶段使用尾帧**：推理时只在前 `τ` 步使用尾帧条件，后续步骤主要负责细节与纹理，让收尾过渡更自然

### 1.4 补充控制信号：Pose 怎么“插进来”

PixelDance 继承了字节系 MagicAnimate/MagicPose 的技术基因，典型用法是把姿态序列当作更强的动作控制信号：

- 输入：参考图（ID）+ 骨架序列（Motion）
- 输出：动作与姿态一致，但角色外观保持参考图风格的视频

## 2. 为什么 80% 难度在数据工程

如果我们要复刻 PixelDance，代码可能只占 20%，**80% 的难度在于数据**。原因很简单：模型是从统计规律里学会“怎么动”的，而统计规律的质量完全由数据决定。

### 2.1 Triplet 是“数据结构”，不是一句口号

你需要把海量视频切成 clip，并构建最小训练样本：

- `start_frame`：clip 的第一帧
- `end_frame`：clip 的末帧（或末尾几帧随机抽一帧）
- `text`：描述“发生了什么变化”的文本

这类样本教给模型的不是“画面是什么”，而是“从 A 变到 B 的运动轨迹应该长什么样”。

### 2.2 “高动态”样本筛选：让模型真的见过足够多的运动

视频数据里“运动”很多是假的：快切、转场、镜头推拉、抖动、字幕贴纸。它们会显著污染时序建模。一个更可操作的清洗流水线可以浓缩成三件事：

1.  **去快切**：先做分镜检测，把快切/转场片段剔除或分桶处理
2.  **选真运动**：用光流/pose 速度做 motion score，把“确实在动”的片段挑出来
3.  **压镜头动**：估计全局相机运动，把“镜头动得太狠”的片段剔除或降权，同时做视觉质量过滤（分辨率、压缩、遮挡、水印等），再把文本与片段对齐（标注或自动 caption + 规则增强动作词）

### 2.3 控制信号的构造：把“想怎么动”变成可学习的输入

为了把不可控的生成问题变成可控的条件生成问题，常见做法是额外构造控制信号：

- **Pose 条件**：OpenPose/DensePose 的序列（人体动作最常用）
- **边缘/草图条件**：Canny/HED（适合结构强约束）
- **语义/深度条件**：分割/深度图（适合几何稳定）

这些控制信号的作用不是让模型“更清晰”，而是把“你希望它怎么动”变成可学习的输入。

## 3. 本系列小结：可控视频的竞争点在“条件化”

PixelDance 想表达的一个方向很清晰：AI 视频生成的下一阶段，不再只比拼“谁生成得更像电影”，而是比拼**“谁能更精准地理解并遵守导演的约束”**。
