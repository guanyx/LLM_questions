### 1️⃣ 概念篇：Claude Skills 是什么？
💡 **核心公式：Skill = MCP Tools + SOP + Domain Context**
*   **Chatbot vs Agent**：以前是“只动口不动手”，现在是“既动口又动手”。
*   **MCP (Model Context Protocol)**：就像 AI 的 **USB 接口**。只要你的数据（数据库、API）适配了这个接口，Claude 就能即插即用。
*   **Skill (技能)**：不仅仅是 API 工具，更是**工具 + 专家经验 (SOP)** 的打包。不仅给 AI 锤子，还教它怎么敲钉子！

### 2️⃣ 逻辑篇：Skill 是怎么运行的？
🔧 **别被代码吓到，其实逻辑超简单！**
*   **Skill 不是脚本，是特种兵**：Claude 是指挥官，Skill 是独立运行、听命行事的特种兵（Micro-server）。
*   **USB 连接 (stdio)**：它们之间不通过 WiFi (HTTP) 慢吞吞地发邮件，而是直接通过 **stdio (标准输入输出)** 像神经连接一样秒速通信。
*   **Don't Crash!**：Agent 开发第一铁律——**Skill 绝对不能死**。遇到错误（如文件找不到）不要崩溃，要返回错误信息，让 Claude 自己想办法修复。

### 3️⃣ 避坑篇：初学者的工程指南
🚧 **从代码到落地的 3 个关键点**
*   **线插在哪？(Config)**：Claude 不会自动扫描。你得去 `claude_desktop_config.json` 里告诉它：“嘿，这里有个 Skill，路径是这个！”
*   **安全第一 (Security)**：
    *   **User in the loop**：Claude 想执行危险操作（如删文件）前，必须经过你点击 [允许]。
    *   **API Key 别裸奔**：Token 不要写死在代码里，要放在配置文件的 `env` 环境变量里。
*   **别造轮子 (Ecosystem)**：想连 GitHub？想连 Postgres？去搜 `mcp-server`，大神们早就写好了，`npm install` 拿来就能用！

### 4️⃣ 实战篇：设计一个“股票监控 Skill”
📈 **架构师视角的 3 步走**
1.  **Resource (账本)**：定义 `stock://portfolio`，让 Claude 能读取你本地 CSV 里的持仓数据。
2.  **Tool (眼睛)**：封装 `get_stock_price` 工具，让 Claude 能查实时股价（因为它自己不能上网）。
3.  **Prompt (大脑)**：注入 `analyze_portfolio` 的 SOP —— “先读账本，再查价格，算出亏损超过 5% 就给我标红报警！”
