# Agent记忆系统怎么实现：第 1 张封面图文生图脚本

Create a highly detailed technical cover infographic for social media in hand-drawn sketch style on pure white background. The theme is "Agent记忆系统怎么实现". The image should immediately communicate that an Agent memory system is not one storage block or one vector database, but a runtime engineering pipeline with write paths, memory layers, recall paths, compaction, conflict handling, and forgetting.

**Design Concept:**

Use a "memory runtime blueprint" metaphor instead of a brain, neural network, or warehouse metaphor. The center of the image should not be a human-like brain or glowing AI head. Because the target canvas is a 3:4 portrait ratio, do NOT build the whole picture as one long horizontal strip. Instead, use a portrait-first composition with one top-to-bottom main structure: inputs at the upper area, write path beneath them, a large central memory zone occupying the middle, recall and context assembly in the lower-middle area, and the final current-context window near the bottom-right or bottom-center. The overall feeling should be: memory is a controlled engineering flow, not a magical storage organ.

**Layout Structure:**

**Top Section (Title Area - 18%):**

- Main title in bold, large font: "Agent记忆系统怎么实现"
- Subtitle below: "它不是一个向量库，而是一条从写入、召回到治理的工程链路"
- Small lead sentence under subtitle:
  "真正难的不是把历史存下来，而是决定什么该写、什么该取、什么该压、什么该忘"
- Typography should feel like a technical editorial cover, centered and clean, not like a futuristic AI poster

**Central Section (Hero System Diagram - 72%):**

Create one dominant portrait-oriented memory-system blueprint optimized for a 3:4 canvas. The central visual should clearly show that memory is made of distinct paths and modules rather than one monolithic storage block.

- Do NOT stretch all stages onto one horizontal axis
- Use a top-to-bottom reading order with local left-right branching only where it improves clarity
- The middle memory zone should be the unquestioned focal area, large and highly legible on mobile
- The extra space should be used to make the center more open, with more breathing room around each memory block and around the recall/governance relationships

- Stage 1 - 输入源区
  - place 5 small input cards across the upper area, not all squeezed onto one line if space is tight
    - user messages
    - tool results
    - files
    - web content
    - error logs
  - all five should converge downward into one intake strip labeled "信息流入"
  - mini annotation: "Agent 的记忆不只来自对话，还来自工具、文件、执行结果和错误反馈"

- Stage 2 - Write Path
  - below the intake strip, place one clearly labeled pipeline section: "Write Path"
  - split this section into three compact internal steps:
    - raw events
    - importance check
    - routing
  - add tiny Chinese annotations:
    - "先记录原始事件"
    - "判断是否值得长期保留"
    - "分流到不同记忆层"
  - make this section feel like the first core engineering gate of the whole image
  - keep this stage moderately compact so it does not steal too much room from the central memory zone

- Stage 3 - 四层记忆系统
  - place this as the dominant middle-area composition
  - after the Write Path, fan the flow into four distinct memory blocks arranged as a portrait-friendly 2x2 grid or a vertical staggered cluster
  - the four blocks should be:
    - Working Set
    - Episodic Memory
    - Semantic Memory
    - Procedural Memory
  - each block should have one compact internal Chinese descriptor:
    - Working Set: "当前工作面"
    - Episodic Memory: "历史事件与回放"
    - Semantic Memory: "稳定事实与偏好"
    - Procedural Memory: "技能与可复用流程"
  - show visually that these four are not equal-size storage cylinders; they are different memory roles with different lifecycles
  - add mini lifecycle hints beside them:
    - 短期
    - 中期
    - 长期
    - 可复用
  - increase the visual footprint of this section so the four layers become the center anchor of the composition rather than a secondary storage cluster
  - show slight differences in shape and density:
    - Working Set feels closer to the current context and more active
    - Episodic Memory feels like searchable historical trails
    - Semantic Memory feels compact, curated, and stable
    - Procedural Memory feels like reusable templates, playbooks, or skill cards
  - the four blocks should feel like a central memory field, not like four tiny storage boxes
  - prioritize readability over symmetry; it is acceptable if one or two blocks are slightly larger to fit the 3:4 page better

- Stage 4 - Recall Path
  - place this below or diagonally below the central memory zone, not all the way to the far right
  - create a second major pipeline labeled "Recall Path"
  - this pipeline should visibly be different from Write Path but structurally parallel in importance
  - split into four steps:
    - task judge
    - retrieve
    - rerank
    - context pack
  - add compact Chinese notes:
    - "先判断这轮要不要搜"
    - "按层召回候选记忆"
    - "按时间、相关性、可信度过滤"
    - "不是贴原文，而是拼状态"
  - visually open up this pipeline so it mirrors Write Path in weight and readability
  - make the "rerank" and "context pack" steps more explicit than before, since this is where the system stops being retrieval and starts becoming usable context
  - if needed, allow this path to bend gently in portrait layout rather than forcing it into one straight line

- Stage 5 - 当前上下文窗口
  - place one focused output box labeled "当前上下文窗口" near the lower-right or bottom-center area
  - make only a small, curated, compressed subset flow into this box
  - inside the box, show 4 tiny structured labels:
    - 稳定事实
    - 关键事件
    - 已验证流程
    - 下一步状态
  - mini annotation: "真正进入 prompt 的，只是当前任务最需要的那部分"
  - this box should be slightly larger and cleaner than before, so it feels like the distilled result of the entire system rather than just a terminal output node
  - give this output area enough quiet white space around it so it reads as the refined destination of the system

- Stage 6 - 记忆治理层
  - place 3 auxiliary memory-governance modules around the outer perimeter of the central memory zone and lower-middle area:
    - Conflict Resolution
    - Compaction
    - Forgetting
  - these modules are not part of the main execution path; they govern the quality, freshness, and usability of memory over time
  - connect them with subtle secondary arrows to the memory blocks first, and then let their influence extend into the Recall Path
  - each module gets one tiny annotation:
    - Conflict Resolution: "新旧事实冲突时，谁算当前态"
    - Compaction: "长历史要被压成高密度状态"
    - Forgetting: "低价值和过期记忆要逐步退出"
  - visually, they should feel like a governance side-layer centered on the memory zone rather than generic floating side cards
  - although they are not main-path steps, their effects should clearly propagate from the memory zone into retrieval quality and final context assembly
  - in portrait layout, these modules can sit around the center like side systems rather than being pushed into one bottom row

- Stage 7 - 一个橙色反例提醒
  - add one compact caution callout in orange near the bottom:
    - label: "常见误解"
    - annotation:
      "把记忆系统等同于聊天记录 + 向量库，通常只会得到更脏、更贵、更难控的上下文"

- Structural emphasis:
  - use one thick blue main arrow network for the dominant flow:
    - inputs -> Write Path -> memory layers -> Recall Path -> current context window
  - use thinner secondary lines for governance:
    - conflict handling
    - compaction
    - forgetting
  - make it visually obvious that "写入" and "召回" are two different engineering paths
  - the whole composition should read like a runtime memory blueprint, not a storage poster
  - portrait readability is more important than forcing one geometric axis
- Add a very small footer label only:
  "Agent记忆系统怎么实现 / 01"

- The image must feel like a memory control plane, not a memory container
