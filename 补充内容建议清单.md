# LLM 知识体系补充建议清单

经过对 `LLM_questions` 目录的遍历与分析，基于当前（2025 年）生成式大模型的主流知识体系、工程实践及最新技术趋势，为您梳理了以下建议补充的内容清单。

## 一、 知识体系缺失 (Knowledge System Gaps)

目前目录在“预训练”和“应用”之间存在明显的断层，即**微调与对齐**环节。这是将基座模型转化为有用产品的关键步骤。

### 1. 微调与对齐 (Fine-tuning & Alignment) - **[高优先级]**

- **SFT (Supervised Fine-Tuning)**
  - 指令微调（Instruction Tuning）的数据构建策略。
  - Chat 格式（ChatML）与 Base 模型的本质区别。
- **RLHF (Reinforcement Learning from Human Feedback)**
  - PPO (Proximal Policy Optimization) 算法在 LLM 中的具体实现步骤。
  - Reward Model (RM) 的训练与打分机制。
- **DPO (Direct Preference Optimization)**
  - **核心议题**：为什么 DPO 正在取代 RLHF？其数学推导如何绕过 Reward Model？
  - **变体**：KTO (Kahneman-Tversky Optimization), ORPO, IPO。
- **PEFT (Parameter-Efficient Fine-Tuning)**
  - **LoRA (Low-Rank Adaptation)**：低秩矩阵分解的数学原理（不仅仅是应用）。
  - **QLoRA**：4-bit 量化下的微调精度保持技术（NormalFloat4 数据类型）。

### 2. 多模态理解 (Multimodal Understanding)

目前目录侧重于扩散模型（图像生成），缺乏对视觉理解（VLM）的覆盖。

- **CLIP (Contrastive Language-Image Pre-training)**：图文对齐的基石，如何连接文本与图像空间。
- **VLM 架构**：
  - **LLaVA 范式**：Visual Encoder + Projector + LLM。
  - **Projector 设计**：Linear vs MLP vs Q-Former。
  - **Visual Token**：图像如何被“翻译”成 LLM 能懂的 Token。

## 二、 工程实践缺失 (Engineering Practice Gaps)

工程部分目前侧重于 RAG 和 Agent (Dify/DSPy)，在模型本身的**推理优化**和**评估**方面较为薄弱。

### 1. 推理与部署优化 (Inference & Deployment)

虽然已有 KV Cache 章节，但现代 LLM 推理系统包含更多优化技术。

- **量化 (Quantization)**
  - **PTQ (Post-Training Quantization)** vs **QAT (Quantization-Aware Training)**。
  - **主流算法**：AWQ (Activation-aware Weight Quantization), GPTQ, SmoothQuant。
  - **精度权衡**：INT8, INT4, FP8 在显存与精度间的取舍。
- **推理加速框架**
  - **vLLM**：PagedAttention 的内存管理机制（操作系统页表思想的迁移）。
  - **Continuous Batching**：如何处理长短不一的请求流。
- **推测采样 (Speculative Decoding)**
  - 利用小模型（Drafter）加速大模型生成的原理。

### 2. 评估体系 (Evaluation)

目录中 `九. 评估与基准` 似为空白，这是闭环迭代的必要环节。

- **基准测试 (Benchmarks)**
  - **MMLU, GSM8K, HumanEval**：它们分别测试模型的什么能力？存在什么局限（数据泄露）？
- **LLM-as-a-Judge**
  - 使用 GPT-4 或专门的 Judge 模型进行自动化评估。
  - Bias 分析：位置偏差（Position Bias）、长度偏差（Length Bias）。
- **数据污染检测**
  - 如何检测模型是否“背过”测试题？

### 3. 数据工程 (Data Engineering)

- **合成数据 (Synthetic Data)**
  - "Textbooks Are All You Need"：如何用 LLM 生成高质量预训练/微调数据。
  - Self-Instruct：让模型自己生成指令数据。
- **数据清洗 Pipeline**
  - MinHash/SimHash 去重算法。
  - PII (个人隐私信息) 去除。

## 三、 最新技术点 (Latest Tech Trends)

针对 2024-2025 年涌现的新技术，建议增加以下专题。

### 1. 推理模型 (Reasoning Models) - **[热点]**

- **DeepSeek-R1 / OpenAI o1**
  - **CoT (Chain of Thought)**：从 Prompt Engineering 进化为训练目标。
  - **Process Reward Model (PRM)**：不仅奖励结果，更奖励推理步骤的正确性。
  - **RL in Reasoning**：强化学习如何提升逻辑推理能力（Aha Moment）。
- **System 1 vs System 2**
  - 快思考（直觉生成）与慢思考（搜索/规划）在模型架构中的体现。

### 2. 长上下文优化 (Long Context)

- **RoPE 变体**
  - **Yarn, LongRoPE**：如何通过插值与外推扩展位置编码。
- **Ring Attention**
  - 分布式训练超长上下文序列的技术。
- **无限注意力**
  - **Infini-attention**：结合 Transformer 与 记忆检索。

### 3. 端侧小模型 (On-device / SLM)

- **MobileLLM**
  - Deep & Thin 架构设计。
  - 层共享（Layer Sharing）与 嵌入共享（Embedding Sharing）。
