我是 Boris，Claude Code 的创建者。我想快速分享一些使用 Claude Code 的技巧，这些技巧直接来源于 Claude Code 团队。团队使用 Claude 的方式与我不同。请记住：使用 Claude Code 没有一种所谓的“正确”方式——每个人的设置都是不同的。你应该通过实验找到适合你的方式！

1. 更多地并行工作

同时启动 3-5 个 git worktree，每一个都并行运行自己的 Claude 会话。这是最大的生产力解锁，也是团队给出的首要建议。就我个人而言，我使用多个 git checkout，但大多数 Claude Code 团队成员更喜欢 worktree——这也是 @amorriscode 在 Claude Desktop 应用中为其构建原生支持的原因！

有些人还会给他们的 worktree 命名并设置 shell 别名（za, zb, zc），这样他们就可以通过一个按键在它们之间跳转。其他人则有一个专门的“分析”worktree，仅用于读取日志和运行 BigQuery。

2. 在计划模式下开始每一项复杂的任务。把你的精力投入到计划中，这样 Claude 就能一次性搞定实现。

有人让一个 Claude 编写计划，然后启动第二个 Claude 作为 Staff Engineer（首席工程师）来审查它。

另一个人说，一旦事情出现偏差，他们就会切换回计划模式并重新计划。不要强行推进。他们还明确告诉 Claude 进入计划模式进行验证步骤，而不仅仅是为了构建。

3. 投资你的 CLAUDE.md。每次修正后，以这句话结尾：“更新你的 CLAUDE.md，这样你就不会再犯同样的错误了。” Claude 在为自己编写规则方面出奇地擅长。

随着时间的推移，无情地编辑你的 CLAUDE.md。不断迭代，直到 Claude 的错误率显著下降。

一位工程师告诉 Claude 为每个任务/项目维护一个笔记目录，并在每次 PR 后更新。然后他们将 CLAUDE.md 指向该目录。

4. 创建你自己的 skill 并提交到 git。在每个项目中复用。

来自团队的提示：

- 如果你每天做某件事超过一次，把它变成一个 skill 或命令
- 构建一个 /techdebt 斜杠命令，并在每次会话结束时运行它，以查找并消除重复代码
- 设置一个斜杠命令，将 Slack、GDrive、Asana 和 GitHub 最近 7 天的内容同步到一个上下文中
- 构建分析工程师风格的 agent，用于编写 dbt 模型、审查代码并在开发环境中测试更改

5. Claude 能自己修复大多数 bug。我们是这样做的：

启用 Slack MCP，然后将 Slack bug 线程粘贴到 Claude 中，只需说“fix”（修复）。无需上下文切换。

或者，只需说“Go fix the failing CI tests”（去修复失败的 CI 测试）。不要微观管理它是如何做的。

让 Claude 指向 docker 日志以排查分布式系统故障——它在这方面出奇地能干。

6. 提升你的提示技巧

a. 挑战 Claude。说“Grill me on these changes and don't make a PR until I pass your test”（针对这些更改盘问我，在我通过你的测试之前不要创建 PR）。让 Claude 做你的审查者。或者，说“Prove to me this works”（向我证明这行得通），让 Claude 比较 main 分支和你的功能分支之间的行为差异。

b. 在一个平庸的修复之后，说：“Knowing everything you know now, scrap this and implement the elegant solution”（基于你现在知道的一切，废弃这个并实现优雅的解决方案）。

c. 在移交工作之前编写详细的规范并减少歧义。你越具体，输出就越好。

7. 终端和环境设置

团队超爱 Ghostty！很多人喜欢它的同步渲染、24 位颜色和正确的 unicode 支持。

为了更轻松地管理 Claude，使用 /statusline 自定义你的状态栏，使其始终显示上下文使用情况和当前 git 分支。我们需要许多人还会对终端标签页进行颜色编码和命名，有时使用 tmux——每个任务/worktree 一个标签页。

使用语音听写。你说话的速度比打字快 3 倍，结果你的提示会变得更加详细。（在 macOS 上按两次 fn 键）

8. 使用子智能体 (subagents)

a. 在任何你希望 Claude 投入更多计算资源的问题后附加“use subagents”（使用子智能体）。

b. 将单个任务分流给子智能体，以保持主智能体的上下文窗口干净且专注。

c. 通过 hook 将权限请求路由到 Opus 4.5——让它扫描攻击并自动批准安全的请求。

9. 使用 Claude 进行数据和分析

让 Claude Code 使用“bq” CLI 实时拉取和分析指标。我们将 BigQuery skill 检入代码库，团队中的每个人都直接在 Claude Code 中使用它进行分析查询。就我个人而言，我已经 6 个多月没写过一行 SQL 了。

这适用于任何具有 CLI、MCP 或 API 的数据库。

10. 用 Claude 学习

团队关于使用 Claude Code 进行学习的一些提示：

a. 在 /config 中启用“Explanatory”（解释性）或“Learning”（学习）输出风格，让 Claude 解释其更改背后的_原因_。

b. 让 Claude 生成一个可视化的 HTML 演示文稿来解释不熟悉的代码。它制作的幻灯片出奇地好！

c. 让 Claude 绘制新协议和代码库的 ASCII 图表，以帮助你理解它们。

d. 构建一个间隔重复学习 skill：你解释你的理解，Claude 提出后续问题以填补空白，并存储结果。
