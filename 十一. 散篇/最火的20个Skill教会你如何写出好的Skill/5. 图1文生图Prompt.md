# 图 1 文生图 Prompt

Create a highly detailed technical infographic in hand-drawn sketch style on a pure white background. The theme is: "Before talking about high quality, first see where an ordinary Skill goes wrong". The audience is experienced programmers who have already written Skills or `SKILL.md` files and want a concrete, practical diagnosis instead of abstract principles.

This is Image 1 in the same 3:4 vertical series as the cover. It should continue the same visual language:

- white background
- engineering review board aesthetics
- hand-drawn technical diagram feel
- dense but controlled information layout
- Chinese-first visible text
- minimal connectors
- no decorative metaphor overload

This page should feel like someone put a real Skill on the table and started marking it up with a red pen.

**Layout Structure:**

**Top Section (Header Area - 13% height):**

- Main title in bold large Chinese font:
  "先别谈高质量，先看一个普通 Skill 到底差在哪"
- Subtitle below in smaller Chinese font:
  "拿一个页面验证 Skill 开刀，直接看它为什么总出问题"
- Keep the header short, direct, and practical

**Section 1 - Bad Example Board (30% height):**

Add a clear Chinese section title:
"先看这份‘不算差，但很不稳’的 Skill"

Create a split board:

- left side shows a simplified bad `SKILL.md` snippet
- right side shows red handwritten review comments

Visible bad-example text on the left:

- "名称：网页助手"
- "描述：帮助用户浏览网页、收集信息、执行操作、检查页面问题"
- "使用时机：当任务和网页、页面、网站、浏览器有关时"
- "执行方式：打开页面，查看内容，按需操作，给出结果"
- "其他：如有问题请重试"

Visible red review comments on the right:

- "名字太泛"
- "没有明确触发词"
- "边界几乎没有"
- "流程只有空话"
- "失败信号被一句‘重试’糊过去了"

Do not make this look like polished marketing copy.
It should feel like a real review markup.

**Section 2 - What Happens In Practice (27% height):**

Add a clear Chinese section title:
"把这种 Skill 丢给 Agent，最容易出现 5 个结果"

Create 5 compact problem cards.
Keep them short.

Visible card texts:

- "该触发的时候不触发"
- "不该触发的时候乱触发"
- "真触发后很容易跑偏"
- "一失败就很难排查"
- "时间一长，文档会越补越肿"

These cards should feel like things people have really seen, not abstract categories.

**Section 3 - Short Diagnosis Band (21% height):**

Add a strong Chinese section title:
"说白了，它不是不能用，是一用就飘"

Create 3 strong conclusion blocks:

- "命中靠运气"
- "执行靠猜"
- "失败被掩盖"

No need to explain too much here.
This band should hit hard and read fast.

**Bottom Section (Conclusion Area - 9% height):**

Place a strong Chinese closing statement:

- "先别急着补内容"
- "先把它为什么不稳看清楚"

**Visual Style:**

- Professional engineering hand-drawn sketch style
- Pure white background with restrained blue-gray drafting details
- Architecture diagnosis aesthetics
- Information-dense but easy to scan
- Strong hierarchy between bad example, red comments, and result cards
- No decorative clutter
- No cartoon elements

**Typography:**

- All visible text inside the image should be Chinese as much as possible
- Only keep mixed-language technical terms when necessary:
  - Skill
  - SKILL.md
- Text should sound like a human reviewer talking
- Avoid robotic labels like "问题 1", "模块 2", or "维度 3"

**Color Palette:**

- Main structural color: deep blue-gray
- Highlight accents: teal
- Warning accents: muted orange
- Risk accents: restrained red
- Background: pure white
- Avoid glossy gradients or neon effects

**Composition Constraints:**

- Vertical 3:4 aspect ratio
- Minimize connector lines
- Use whitespace to separate the major blocks
- Keep the reading path top -> example -> consequences -> conclusion
- Around 48% structured blocks/cards, 22% text, 30% whitespace

**Negative Guidance:**

- Do not generate a generic AI infographic
- Do not use radial webs, dense arrows, or messy connector networks
- Do not use robots, human characters, mascots, or sci-fi effects
- Do not turn the image into a long flowchart
- Do not make it feel like a beginner tutorial page

Style: white-background technical hand-drawn engineering infographic, red-pen review board, Chinese-first typography, minimal connectors, practical diagnosis layout for experienced programmers
