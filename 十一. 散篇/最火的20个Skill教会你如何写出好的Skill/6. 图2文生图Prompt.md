# 图 2 文生图 Prompt

Create a highly detailed technical infographic in hand-drawn sketch style on a pure white background. The theme is: "How to actually fix the name, description, boundaries, and workflow of a Skill". The audience is experienced programmers who have already written Skills or `SKILL.md` files and now want to turn an unstable Skill into one that triggers more accurately and runs in a more predictable way.

This is Image 2 in the same 3:4 vertical series. It should continue the same visual language:

- white background
- engineering review aesthetics
- hand-drawn technical diagram feel
- dense but disciplined layout
- Chinese-first visible text
- minimal connectors
- no decorative clutter

This page should feel like a real revision sheet.
Not theory.
Not a checklist poster.
A real before-and-after rewrite.

**Layout Structure:**

**Top Section (Header Area - 12% height):**

- Main title in bold large Chinese font:
  "改 Skill，第一步不是补案例，而是把这 4 块改对"
- Subtitle below in smaller Chinese font:
  "先把名字和描述改准，再把边界和流程写实"
- Keep the header clean, sharp, and easy to scan

**Upper Left - Rename And Rewrite (24% height):**

Add a Chinese section title:
"先把任务缩窄：从‘网页助手’改成‘页面验证’"

Create a clean before-vs-after comparison block.

Visible text:

- "改前"
  "名称：网页助手"
  "描述：帮助用户浏览网页、收集信息、执行操作、检查页面问题"

- "改后"
  "名称：页面验证"
  "描述：打开本地或测试环境页面，检查布局错位、明显报错、关键按钮状态和截图结果；当任务涉及‘验证页面是否正常’‘检查改动有没有把页面弄坏’‘对比页面表现’时使用"

Add two short handwritten-style review notes nearby:

- "名字只留一个动作"
- "description 要把做什么和什么时候用一起写出来"

**Upper Right - Boundary Block (24% height):**

Add a Chinese section title:
"想少误触发，别只写 Use When，先把排除条件写出来"

Create a strong boundary card.

Visible text:

- "不要在这些场景使用"
- "需要长期浏览和整理网页资料时"
- "需要批量采集站点内容时"
- "重点是修改代码而不是验证页面时"
- "需要真实登录、支付、提交敏感数据时"

Add two short notes:

- "边界一虚，就会乱触发"
- "先写窄一点，比一上来追求大而全更稳"

**Lower Main Section - Workflow Rewrite (28% height):**

Add a Chinese section title:
"流程不要写成空话，要写成检查链"

Create a 5-step workflow chain.
Keep the connectors minimal and short.
Do not turn this into a giant flowchart.

Visible step texts:

- "先确认访问目标，是本地地址、测试环境还是指定 URL"
- "再等页面稳定，不在首屏还没渲染完时下结论"
- "然后看有没有明显报错、空白区域、关键模块缺失"
- "再看关键交互元素能不能看见、能不能点、状态对不对"
- "最后截图并给结论，写清看到了什么问题，或者没看到什么问题"

Add two small review notes nearby:

- "没有检查点，后面就很难查"
- "没有停止条件，很容易越做越歪"

**Bottom Section (Conclusion Area - 12% height):**

Place a strong Chinese closing statement:

- "先修名字、描述、边界、流程"
- "这样一个 Skill 才会慢慢稳下来"

**Visual Style:**

- Professional engineering hand-drawn sketch style
- Pure white background with restrained blue-gray drafting details
- Architecture review aesthetics, not pop infographic aesthetics
- Strong hierarchy: title -> rewrite blocks -> workflow -> bottom conclusion
- Dense but clean
- Use cards, before/after blocks, compact notes
- Avoid decorative metaphors

**Typography:**

- All visible text inside the image should be Chinese as much as possible
- Only keep mixed-language technical terms when necessary:
  - Skill
  - Use When
  - URL
- Titles should be short and natural
- The wording should sound like a human technical reviewer, not an autogenerated manual

**Composition Constraints:**

- Vertical 3:4 aspect ratio
- Avoid dense connector lines
- Use spacing, alignment, and zoning instead of arrows
- If any connector is used, keep it to a few short structural dividers only
- The image should be readable in one pass
- Around 54% structured blocks, 18% text, 28% whitespace

**Color Palette:**

- Main structural color: deep blue-gray
- Highlight accents: teal
- Warning accents: muted orange
- Risk accents: restrained red
- Background: pure white
- Avoid glossy surfaces and heavy gradients

**Negative Guidance:**

- Do not generate a generic AI infographic
- Do not use spider-web diagrams or radial spoke layouts
- Do not use robots, mascots, human characters, or sci-fi effects
- Do not turn the whole image into a complex flowchart
- Do not make it look like a school handout
- Do not overload the image with arrows or callout lines

Style: white-background technical hand-drawn engineering infographic, rewrite sheet, before-and-after review board, Chinese-first typography, minimal connectors, dense but disciplined layout for experienced programmers
