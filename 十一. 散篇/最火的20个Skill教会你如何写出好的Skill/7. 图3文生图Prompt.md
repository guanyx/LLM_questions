# 图 3 文生图 Prompt

Create a highly detailed technical infographic in hand-drawn sketch style on a pure white background. The theme is: "What people most often forget when they try to make a Skill production-ready". The audience is experienced programmers who have already built Skills or `SKILL.md` files and now want to understand how to make results actually finish, how to expose failures early, and how to keep permissions under control.

This is Image 3 in the same 3:4 vertical series. It must continue the same design language:

- white background
- engineering review board aesthetics
- hand-drawn technical diagram look
- dense but disciplined information layout
- Chinese-first visible text
- minimal connectors
- no decorative metaphor overload

This page should feel like the moment where a review goes from “looks okay” to “wait, three important things are still missing”.

**Layout Structure:**

**Top Section (Header Area - 12% height):**

- Main title in bold large Chinese font:
  "很多 Skill 看起来执行完了，其实根本没完成"
- Subtitle below in smaller Chinese font:
  "最容易漏的，偏偏最要命：完成定义、失败信号、权限约束"
- Keep the header short, direct, and practical

**Upper Left - Completion Definition (24% height):**

Add a Chinese section title:
"先补完成定义：怎么才算真的做完"

Create a clean completion-definition card.

Visible text:

- "页面已成功打开"
- "至少完成一轮首屏检查"
- "对关键模块或关键按钮做了状态确认"
- "产出一张截图"
- "结论中明确写出‘发现的问题’或‘未发现明显问题’"

Add one short side note:

- "没有完成定义，你只能说做过了，不能说做完了"

**Upper Right - Failure Signals (24% height):**

Add a Chinese section title:
"再补失败信号：什么时候停，什么时候追问"

Create 4 compact failure-signal cards.

Visible text:

- "页面打不开：地址不可达、服务没启动、网络被拦截"
- "页面始终不稳定：长时间加载、反复重绘、关键元素迟迟不出现"
- "页面需要登录或敏感权限：没有凭证时不要硬闯"
- "当前任务已经不是验证页面，而是修代码、抓数据或提交真实操作"

Add one short side note:

- "别再写‘如有问题请重试’这种空话"

**Lower Main Section - Permission And Reference Split (28% height):**

Add a Chinese section title:
"最后再补权限和资料怎么放"

Create a clean 3-column restriction board.

Visible texts:

- "可以做"
  "打开测试页面"
  "点击不提交真实数据的界面元素"
  "截图"
  "读取前端报错"

- "不要做"
  "登录真实账号"
  "执行支付"
  "提交表单到生产环境"
  "修改后端数据"

- "需要追问"
  "测试地址"
  "账号权限"
  "目标页面列表"

Add one compact bottom note strip:

- "入口只留最小流程和关键约束"
- "重资料别塞入口，放外面"

**Bottom Section (Conclusion Area - 12% height):**

Place a strong Chinese conclusion block:

- "把完成定义、失败信号、权限约束补齐"
- "这个 Skill 才像个能上线的东西"

**Visual Style:**

- Professional engineering hand-drawn sketch style
- Pure white background with restrained blue-gray drafting details
- Architecture review aesthetics, not pop infographic aesthetics
- Strong hierarchy between the three missing pieces
- Dense but still readable in one pass
- Use clean cards, compact tables, and review tiles
- Avoid decorative metaphors and playful illustrations

**Typography:**

- All visible text inside the image should be Chinese as much as possible
- Only keep English or mixed-language technical terms when necessary:
  - Skill
- Titles should sound natural, like a human technical reviewer talking
- Avoid robotic labels like "模块 1", "模块 2", or "关键点 3"

**Composition Constraints:**

- Vertical 3:4 aspect ratio
- Keep the upper and lower sections clearly separated
- Avoid dense connector lines between the three major blocks
- If any connector is used, keep it to a few short structural dividers only
- Use alignment, whitespace, and grouping to organize the page
- Around 53% structured blocks, 19% text, 28% whitespace

**Color Palette:**

- Main structural color: deep blue-gray
- Highlight accents: teal
- Warning accents: muted orange
- Risk accents: restrained red
- Background: pure white
- Avoid heavy gradients, glossy effects, and neon glow

**Negative Guidance:**

- Do not generate a generic AI infographic
- Do not use robots, human characters, mascots, or sci-fi effects
- Do not turn the page into a classroom poster
- Do not make the restriction area look like a legal compliance form
- Do not overload the image with arrows or callout lines
- Do not use radial spokes or spider-web structures

Style: white-background technical hand-drawn engineering infographic, missing-pieces review board, Chinese-first typography, minimal connectors, dense but disciplined layout for experienced programmers
