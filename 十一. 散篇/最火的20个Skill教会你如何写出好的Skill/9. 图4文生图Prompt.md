# 图 4 文生图 Prompt

Create a highly detailed technical infographic in hand-drawn sketch style on a pure white background. The theme is: "After rewriting a Skill, how do you test it, fix what still fails, and reuse the method next time". The audience is experienced programmers who have already rewritten at least one Skill and now want a practical closing step: how to verify it, where to patch it, and how to turn the process into a reusable writing order.

This is Image 4 in the same 3:4 vertical series. It must continue the same visual language:

- white background
- engineering review board aesthetics
- hand-drawn technical diagram look
- dense but disciplined information layout
- Chinese-first visible text
- minimal connectors
- no decorative metaphor overload

This page should feel like the last review board in the series:

- first run the minimum acceptance test
- then know exactly where to go back and fix
- finally leave with a reusable skeleton for the next Skill

**Layout Structure:**

**Top Section (Header Area - 12% height):**

- Main title in bold large Chinese font:
  "改完不等于改对，最后还要过验收"
- Subtitle below in smaller Chinese font:
  "先拿最小测试集过一遍，再把这套写法用到下一个 Skill"
- Keep the header clear, calm, and practical

**Upper Left - Minimum Acceptance Flow (26% height):**

Add a Chinese section title:
"验收不用搞复杂，先这样测"

Create a compact 6-step acceptance flow.
Keep connectors short and restrained.

Visible step texts:

- "找 10 个真的应该触发这个 Skill 的任务"
- "再找 10 个看起来像，但其实不该触发的任务"
- "看它会不会在第一组里稳定进场"
- "看它会不会在第二组里克制退出"
- "看它能不能稳定跑到完成定义"
- "看失败时，能不能暴露真实原因，而不是继续瞎做"

**Upper Right - Where To Patch If It Fails (26% height):**

Add a Chinese section title:
"没过的话，回头改哪一块"

Create a problem-to-fix mapping board.

Visible texts:

- "该触发时经常不触发"
  "回头改：名称、description、触发词"

- "不该触发时乱进场"
  "回头改：Don't Use When、边界定义"

- "执行中老是跑偏"
  "回头改：流程、检查点、停止条件"

- "看起来做完了，但结果很虚"
  "回头改：完成定义、输出要求"

- "一失败就只会重试"
  "回头改：失败信号、追问条件、换策略条件"

This section should feel very practical.
The reader should know exactly where to go back.

**Lower Main Section - Reusable Skeleton (24% height):**

Add a Chinese section title:
"下一次，直接按这个顺序写"

Create a reusable skeleton board with 8 short steps.

Visible texts:

- "先写名字：只写一种核心动作"
- "再写描述：把做什么和什么时候用一起写清"
- "再写触发条件：给关键词和典型任务"
- "再写排除条件：把最容易误触发的任务写出来"
- "再写输入输出：开始前要什么，结束时交付什么"
- "再写执行流程：每一步看什么，在哪停，什么情况下继续"
- "再写失败信号：什么时候停，什么时候追问，什么时候换策略"
- "最后补权限约束和外置资料"

Add one short side note:

- "先修命中，再修执行，最后再补治理"

**Bottom Section (Conclusion Area - 12% height):**

Place a strong Chinese conclusion block:

- "高质量 Skill 不是写得更长"
- "而是让命中、边界、流程、失败都能查"
- "能复用的不是一段话，是一套改稿顺序"

**Visual Style:**

- Professional engineering hand-drawn sketch style
- Pure white background with restrained blue-gray drafting details
- Architecture review aesthetics, not pop infographic aesthetics
- Strong hierarchy from acceptance -> fix mapping -> reusable skeleton -> conclusion
- Dense but still readable in one pass
- Use clean cards, compact strips, and review tiles
- Avoid decorative metaphors and playful illustrations

**Typography:**

- All visible text inside the image should be Chinese as much as possible
- Only keep English or mixed-language technical terms when necessary:
  - Skill
  - description
  - Don't Use When
- Titles should sound natural, like a real reviewer closing a review
- Avoid robotic labels like "步骤 1", "步骤 2", or "模块 A"

**Composition Constraints:**

- Vertical 3:4 aspect ratio
- Keep the upper and lower sections clearly separated
- Avoid dense connector lines between the major blocks
- If any connector is used, keep it to a few short structural dividers only
- Use alignment, whitespace, and grouping to organize the page
- Around 54% structured blocks, 18% text, 28% whitespace

**Color Palette:**

- Main structural color: deep blue-gray
- Highlight accents: teal
- Warning accents: muted orange
- Risk accents: restrained red
- Background: pure white
- Avoid heavy gradients, glossy effects, and neon glow

**Negative Guidance:**

- Do not generate a generic AI infographic
- Do not use robots, human characters, mascots, or sci-fi effects
- Do not turn the acceptance flow into a giant pipeline chart
- Do not make the page look like a classroom worksheet
- Do not overload the image with arrows or callout lines
- Do not use radial spokes or spider-web structures

Style: white-background technical hand-drawn engineering infographic, acceptance-and-reuse review board, Chinese-first typography, minimal connectors, dense but disciplined layout for experienced programmers
