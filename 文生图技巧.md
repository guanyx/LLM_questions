## 封面图

```markdown
# 任务

生成一个跟参考图风格和样式都很相似的封面图

标题： RAG 全流程拆解（Query → 检索 → 重排 → 生成）

背景流程步骤： 从用户的 Query 开始，串起检索增强生成（RAG）在一次请求里真正发生的链路：Query 规整与约束 → 向量化（embedding）→ 向量库召回（retrieval）→ 重排（rerank）→ prompt 组装 → 生成（generation）

# 风格定义

## 7. 回答决策（在“能生成”之外，决定“是否该生成”）

1. 证据充足性判断（可选但很关键）：当证据不足或冲突时，选择拒答、追问或提示不确定性（为了避免“没证据也硬答”；就像你资料没找到，最好先承认缺口而不是乱写）
2. 触发二次检索（可选）：当首轮检索命中率低或 Query 太宽，改写 Query 或拆分子问题再检索（为了把“找不到/找不准”变成可恢复的流程；就像第一家书店没找到，你换关键词或换一家再找）
3. 结构化输出约束（可选）：要求回答按固定结构输出，并明确引用位置（为了让答案更可控、更易验；就像用模板写报告，结论、依据、风险分别放在固定位置）
```
