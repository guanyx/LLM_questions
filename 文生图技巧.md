## 当前最新的文生图方式

模型： gpt-image-2

1. 先选题
2. 根据选题编写文章内容
3. 把文章内容丢给模型，生成 封面+分镜 的脚本
4. 再把脚本丢给模型，按照[信息图脚本师](信息图脚本师.md)的Prompt结构，逐张生成Prompt

## 模型

nano banana pro

## 哪里可以用

[YouMind](https://youmind.com/invite/6T5Q8V)可以多轮对话调试图片，整体效果挺不错的，唯一的缺点就是有点贵

所以当你调试出挺不错的图片后，可以调用 nano banana 的接口去生图，便宜很多。[GMI Cloud](https://52237.mxiaoma.com/tw/bfZZ)可以免费试用

[APIYI](https://api.apiyi.com/register/?aff_code=9Wfu) 的生图体验更快，价格更便宜

不推荐 openRouter，图片尺寸会被阉割，导致更容易乱码，图片不清晰

## 信息图

可以单独作为Prompt前缀，也可以设计成一个Agent[信息图脚本师](信息图脚本师.md)。如果你的输入跟我的不同，记得自己调节Agent的指令内容

预期输入的文本示例：

```markdown
## 1. 输入：从 logits 得到“可比较的候选集合”

1. 生成候选（一次或多次）：从 `logits` 经过解码（采样/贪心/约束解码）得到候选答案 `candidates = [c1, c2, ...]`
2. 标准化候选表示：把候选按统一格式表达（比如：最终回答 + 引用/依据 + 假设/前提 + 关键步骤）
3. 记录生成上下文：保留输入、检索片段、工具输出、采样参数等元信息

提示：如果你只生成一个候选、且不记录上下文，那“可靠性”就很容易退化成“相信模型的一次即兴发挥”。

```

## 封面图

生成封面图的时候，YouMind用地还不是很熟练，我最满意的这一张Prompt怎么都不能还原了

勉强搞了个类似的，提示词可以参考[LLM 推理流程（Prompt → 第一个 Token）](LLM%E6%8E%A8%E7%90%86%E6%B5%81%E7%A8%8BPrompt.md)

更简单的方式，还是通过参考图

```markdown
# 任务

生成一个跟参考图风格和样式都很相似的封面图

标题： RAG 全流程拆解（Query → 检索 → 重排 → 生成）

背景流程步骤： 从用户的 Query 开始，串起检索增强生成（RAG）在一次请求里真正发生的链路：Query 规整与约束 → 向量化（embedding）→ 向量库召回（retrieval）→ 重排（rerank）→ prompt 组装 → 生成（generation）
```

## 其它说明

目前我生成的图片，都是选的 2K 分辨率。分辨率低，图片中的中文字号较小时，容易出现乱码

用YouMind进行调试生图时，可以让它返回 *提交给文生图模型的prompt*，可以基于它返回的prompt，去训练自己的生图Agent
